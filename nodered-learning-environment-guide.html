学習環境ガイド：どこで動かす？

📖 このページの役目各ノードを学ぶとき、共有クラウドのNode-REDでやるのか、ラズパイや自分のPCでやるのか。その使い分けの地図を示します。多くはクラウドのまま学べますが、通信やハードの一部は環境を変えた方がうまくいきます。

🎯 得られるもの「このノードはどこで動かすべき？」が一目で分かる環境マップと、その理由、ラズパイやローカルへ移る判断基準を示します。サイト各所に付く環境バッジ（☁️/⚠️/🖥️/🍓）の意味もここで定義します。

➡️ 次に読むページ各ノードガイド ／ 通信の基礎 ／ Raspberry Pi インストール

このサイトの学習は、まず1台の共有クラウドサーバー（学習者ごとにポート違いのNode-RED）で進めます。手軽で、準備が要らず、ほとんどのノードはこれで十分に学べます。

ただし、通信を待ち受ける系のノードや物理センサーを扱う段になると、共有クラウドでは不都合が出ます。HTTP・WebSocketのように、公式サンプルが localhost:1880（自分自身）への接続を前提とするガイドも同様です。次の2層モデルがその切り分けを示します。

1. 2 層モデル：クラウドで進み、必要になったらラズパイ／ローカルへ

☁️

第1層：共有クラウド（できるところまで）

概念、ロジック、データ変換、ダッシュボード、MQTT など全体の約8割はここで完結します。追加の準備は不要です。

⬇ HTTP・WebSocket、通信の待受、物理ハードが必要になったら
🍓🖥️

第2層：ラズパイ／ローカルPC（その先）

HTTP・WebSocketの公式サンプル、TCP/UDPの自由な実験、MODBUS実機、ブロードキャスト、GPIOやセンサー。ラズパイが本命で、WindowsのローカルPCでも代用できます。

大事な前提：これは「場所の分担」ではなく「できることの入れ子」

誤解しやすい点を先に挙げます。第1層と第2層は排他的な縄張りではありません。ローカルPCやラズパイは、クラウドでできることをそのまま実行でき、その上に物理ハードなどが加わります。逆に、クラウドはハードやブロードキャストを扱えません。関係を図にすると次のようになります。

🍓 ラズパイ＝ ローカルPCの全部 ＋ GPIOやSense HATなどオンボードの物理ハード
🖥️ ローカルPC＝ クラウドの全部 ＋ 自由なポート、ブロードキャスト、USBシリアル機器、GUIシミュレータ
☁️ クラウド＝ 概念、ロジック、データ変換、MQTT、Dashboard（全体の約8割）
⬆ 上の段は、下の段でできることをすべて含みます（上位互換）
ヒント：自分のPCやラズパイだけで全部やってもOK
手元のローカルPCやラズパイにNode-REDを入れれば、このサイトの全カリキュラムをそこだけで学べます。共有クラウドは手軽に始められる共通の場にすぎません。クラウドで進めて、必要になった所（HTTP・WebSocket、ハード、自由なポートなど）だけローカルやラズパイに移ると、準備の手間が最小になります。
ヒント：ひとことで
読み物やロジック、画面（Dashboard）、MQTTはクラウドで手軽に。HTTP・WebSocket、通信の待受、ハードはラズパイやローカルで。迷ったら、この後の環境マップを見てください。

2. 環境バッジの凡例（サイト共通の印）

各ノードガイドやトップページのカードに、次の4つのバッジのいずれかが付きます。意味の基準はこのページです。バッジは「最低どの環境があれば動くか」を示す印です。たとえば ☁️ はどこでも（クラウドでも）動き、🍓 はラズパイが要るという意味で、上位環境（ローカルPCやラズパイ）では下位バッジのものも動きます。

☁️ クラウドOK
共有クラウドのNode-REDだけで完結。追加準備は不要。
⚠️ クラウド可（条件付き）
動くが、自分専用のポート番号を使うなどの配慮が必要。
🖥️ ローカル/ラズパイ
クラウドは不可。自分のPCかラズパイで（どちらでも可）。
🍓 ラズパイが必要
物理センサーやGPIOなど。ローカルPCでは代用不可（USB機器を除く）。

4. なぜ分かれるのか（4 つの理由）

4-1. ポートの取り合い（共有サーバーは港が 1 つ）

1台のサーバーに複数のNode-REDが同居しています。エディタのポート（1880, 1881…）は学習者ごとに分かれていますが、TCPで9000番を待ち受けるようなフローは、サーバー全体で1つしかない9000番を奪い合います。先にデプロイした人だけが成功し、他はエラー（アドレス使用中）になります。

対策：TCP/UDP/MODBUSの待受は自分専用のポート番号を使います（講師から割り当てられた番号など）。自由に試したいならローカルやラズパイへ移ります。

4-2. 特権ポート（502 番などは管理者専用）

1024番より小さいポート（MODBUSの502、HTTPの80など）は、安全のため管理者権限がないと開けません。共有クラウドのNode-REDは一般ユーザーで動くため、502番の擬似スレーブは立てられません。

対策：シミュレータは10502のような大きい番号を使います。自分のPC（管理者）なら502もそのまま使えます。

4-3. ブロードキャスト（クラウドでは飛ばない）

UDPの「同じネットワークの全員へ一斉送信」は、家庭や社内のLANでは機能しますが、クラウドのネットワークは通常ブロードキャストを遮断します。機器の自動発見などはクラウドでは再現できません。

対策：ブロードキャスト系の演習は実LAN（ラズパイやローカルPC）で行います。

4-4. 物理ハードウェア（VM にピンは生えていない）

温度センサー、RS-485、GPIO、Sense HAT。これらは現実の配線が必要です。クラウドの仮想マシンには物理的な端子がありません。

対策：ハード演習はラズパイが必須です（USB機器ならローカルPCでも一部可）。

5. クラウドを上手に使うお作法

第1層（共有クラウド）で気持ちよく学ぶための、3つの心得です。

  • 自分のポートを使う：TCP/UDP/MODBUSの待受番号は、割り当てられた自分専用の番号に。サンプルの番号をそのまま全員が使うと衝突します。
  • MQTTのトピックに自分の名前を付ける：共有ブローカーでは test/temp ではなく あなたの名前/test/temp のように。他の学習者と混信しなくなります。
  • Exec・File は控えめに：共有OS・共有ディスクを触ります。システムを変えるコマンドや、他人と同じファイルパスへの書き込みは避けましょう。

6. ラズパイ／ローカル PC へ移るタイミングと準備

次のテーマに進むときが、第2層へ移るサインです。

こうなったら移る先準備の入口
TCP/UDP をポート気にせず自由に実験したい 🖥️ ローカル / 🍓 ラズパイ Windows / Raspberry Pi インストール
MODBUS をシミュレータで手軽に試したい 🖥️ ローカルPC（ModRSsim2） Windows インストール ／ 演習環境ガイド
MODBUS を実機（PLC・電力計）とつなぎたい 🍓 ラズパイ＋RS-485 Raspberry Pi インストール ／ 演習環境ガイド
センサーやGPIOなど物理ハードを扱いたい 🍓 ラズパイ Raspberry Pi インストール ／ Pi Sense HAT
UDPブロードキャストを試したい 🍓 / 🖥️ 実LAN 同一LAN上のラズパイ or ローカルPC
ポイント：ラズパイが本命、ローカル Windows は代用
産業やIoTのリアルさ（実センサー、RS-485、GPIO）まで体験するならラズパイが最適です。手元にPiがなくても、MODBUSシミュレータやTCP/UDPの自由演習はWindowsのローカルPCで代用できます。ハード演習だけは実機（Pi）が要ります。

7. まとめ：環境の地図

  • ローカルPCやラズパイはクラウドの上位互換。クラウドでできることはそのまま実行でき、その上に物理ハードなどが加わる（クラウド ⊂ ローカル ⊂ ラズパイ）。
  • 学習は共有クラウドで進められるところまで進む（全体の約8割）。環境を分けるのは縄張りではなく、準備の手間を最小化するため。
  • クラウドが苦手なのは、通信の待受（TCP/UDP/MODBUS）、ブロードキャスト、物理ハード、そして公式サンプルが localhost:1880 への接続を前提とするガイド（HTTP・WebSocket）の4つ。
  • 待受系は自分専用ポートでクラウド継続も可。自由にやるならローカルやラズパイへ。
  • ハードはラズパイ必須。シミュレータ中心ならローカルPCでも代用可。
  • 各ガイドの環境バッジ（☁️/⚠️/🖥️/🍓）で、その場で判断できます。
