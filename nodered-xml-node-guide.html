📖 1. 概要

XML ノードは、XML文字列とJavaScriptオブジェクトを相互変換するノードです。XML形式のデータを扱うAPIやファイルとの連携に使用します。

📚 翻訳者に例えると：XML ノードは2つの言語（XML形式とJavaScriptオブジェクト形式）を相互に翻訳する通訳者のようなものです。どちらの形式でデータを受け取っても、もう一方の形式に変換してくれます。

XML文字列 → XML → オブジェクト

💡 ポイント:

  • 入力がXML文字列 → オブジェクトに変換
  • 入力がオブジェクト → XML文字列に変換
  • 自動で変換方向を判定
  • 属性やテキストノードも適切に処理

⚙️ 2. 設定詳細

プロパティ

プロパティ説明デフォルト
名前ノードの名前空
プロパティ変換対象のプロパティmsg.payload
XMLタグ属性を次のプロパティ名として表現XML属性を格納するプロパティ名$ (デフォルト)
文字列を参照するための接頭辞テキストコンテンツを格納するプロパティ名_ (デフォルト)

🔄 3. 変換ルール

XML → オブジェクト変換

入力（XML）

<person id="1"> <name>田中太郎</name> <age>30</age> </person>

出力（オブジェクト）

{ "person": { "$": { "id": "1" }, "name": ["田中太郎"], "age": ["30"] } }

オブジェクト → XML変換

入力（オブジェクト）

{ "book": { "$": { "isbn": "123" }, "title": "Node-RED入門", "price": 2000 } }

出力（XML）

<?xml version="1.0"?> <book isbn="123"> <title>Node-RED入門</title> <price>2000</price> </book>

属性とテキストの表現

XML要素オブジェクト表現
属性 <tag attr="value">{ "$": { "attr": "value" } }
テキスト <tag>text</tag>{ "tag": ["text"] } または { "_": "text" }
属性+テキスト{ "$": { "attr": "v" }, "_": "text" }

📝 5. 演習問題

演習1: 基本的なXML変換 初級

📋 課題: シンプルなXML文字列をオブジェクトに変換してください。

✅ 成功の条件:

  • デバッグパネルに表示されるpayloadがオブジェクト型（Object）になっている
  • msg.payload.user プロパティが存在し、その中に name と email が含まれている
  • 各値が配列として格納されている（例: msg.payload.user.name[0] === "山田太郎"）
  • XMLパース中にエラーが発生していない

演習4: 属性付きXMLの生成 上級

📋 課題: 属性を持つXMLをオブジェクトから生成してください。

✅ 成功の条件:

  • デバッグパネルに文字列型（string）のXMLが出力される
  • <response status="success" code="200"> のようにルート要素に属性が付与されている
  • <user id="U001"> のようにネストした要素にも属性が付与されている
  • 生成されたXMLが有効なXML形式になっており、ブラウザのXMLパーサーでエラーなく読み込める
💡 ヒント

✅ 6. まとめ

🎯 重要ポイント:

  • XML文字列 ↔ オブジェクトの相互変換
  • 属性は $ プロパティで表現
  • テキストコンテンツは _ または配列で表現
  • 入力形式を自動判定して変換方向を決定

⚠️ 注意事項:

  • 不正なXMLはエラーになる（事前にCatch設置を推奨）
  • 要素の順序は保持されない場合がある
  • 名前空間を持つXMLは追加の考慮が必要
