📈 1. データ可視化ウィジェット概要

🎨 日常のアナロジー：車のダッシュボード

車を運転するとき、ダッシュボードを見ると様々な情報が視覚的に表示されています：

  • スピードメーター（ゲージ） - 現在の速度を針や数値で表示
  • 燃料計（ゲージ） - 残量をバーやセグメントで表示
  • 走行距離計（テキスト） - 累計距離を数値で表示
  • 燃費グラフ（チャート） - 時系列での変化を折れ線で表示

Dashboard 2.0のデータ可視化ウィジェットも同様に、IoTセンサーやシステムの状態を直感的に把握できる形で表示します。

🎯

gauge

メーター形式の値表示

📋

table

データの一覧表示

📊

progress

進捗状況の表示

📈 3. chart（チャート）

表示 chart ウィジェット

時系列データや比較データを視覚的に表示するチャートウィジェットです。eChartsライブラリをベースに、折れ線グラフ、棒グラフ、散布図、円グラフ、ヒストグラムなど多彩な表現が可能です。

🔹 対応チャートタイプ

📈

Line（折れ線）

時系列データの推移表示に最適

📊

Bar（棒グラフ）

カテゴリ別の比較に最適

⚬

Scatter（散布図）

2変数の相関分析に最適

🥧

Pie/Doughnut（円）

構成比の表示に最適

📶

Histogram

度数分布の表示に最適

🔹 入力データ形式

パターン1: シンプルな数値（時系列自動）

📤 送信データ
msg.payload = 25.5; msg.topic = "温度";

X軸のタイプが「Timescale」で、X値を指定しない場合、現在時刻が自動的に使用されます。

パターン2: オブジェクト形式

📤 送信データ
msg.payload = { "time": 1703500800000, // ミリ秒タイムスタンプ "value": 25.5 }; msg.topic = "センサーA";

パターン3: 配列形式（一括データ）

📤 送信データ
msg.payload = [ { "category": "1月", "sales": 100 }, { "category": "2月", "sales": 150 }, { "category": "3月", "sales": 120 } ];

🔹 補間方法（Interpolation）

折れ線グラフでポイント間をどう描画するかを選択できます：

  • Linear - 直線で接続（デフォルト）
  • Step - 階段状に接続（ON/OFF状態の表示に最適）
  • Bezier - 滑らかな曲線で接続
  • Cubic - より滑らかな曲線
  • Cubic-Mono - オーバーシュートしない滑らかな曲線

🔹 タイムスタンプのフォーマット

フォーマット 表示例
{HH}:{mm}:{ss} 12:30:45
{HH}:{mm} 12:30
{yyyy}-{M}-{d} 2024-12-25
{d}/{M} 25/12
{ee} {HH}:{mm} Wed 12:30

🔹 既知の不具合・制限事項

⚠️ ui-chart には現時点で以下の既知の不具合・注意点が報告されています（FlowFuse Dashboard 2.x 系、eCharts ベース）。

長時間稼働や複雑な構成で利用する場合は、特に下表のワークアラウンドを設計に取り入れてください。

分類 現象 影響 暫定対策
複数系列 複数 series を含むリアルタイム payload を順次送ると、ブラウザ更新やページ切替後にチャート内容が消える。 監視画面・時系列表示で、リロード後に履歴表示が空になる。 更新・ページ切替後に過去データを再送する設計、または一時的に series ごとにメッセージを分割。
動的オプション 毎サンプルごとに msg.ui_update.chartOptions を送ると、内部の dynamicChartOptions 配列が増え続ける。 長時間稼働で CPU 使用率増加、画面遅延、UX 悪化。 chartOptions は毎回送らず、初期化時・変更時のみ送る。必要に応じて throttle / rate limit を入れる。
series オプション msg.ui_update.chartOptions.series の設定がページ更新で永続しない。データ到着前や非表示ページのチャートに送ると反映されないことがある。 線色・系列別設定が意図どおり残らない。 データ投入後に series オプションを再送する。series には name と type を必ず明示する。
legend 必要な series / value キーを持たないデータが混じると、凡例に undefined が表示される。 凡例が汚れる。不要データが内部で保持・再送される可能性。 chart ノードへ送る前に function ノードで series / value が欠けたデータを除外する。
X 軸 棒グラフなどで重複する X ラベルを使うと、後続データが前のデータを上書きするように見える。 「今日 00–23時」「明日 00–23時」のような重複ラベル表示で不都合。 内部 X 値は一意にし、表示ラベルだけ短縮する。eCharts の formatter を活用。
極端な数値 Number.MAX_VALUE や極端値を含むと Y 軸の自動スケールで描画不能となり、タブクラッシュに至るケースもある。 センサー異常値や桁あふれで画面が壊れる可能性。 chart 投入前に値を検証し、上限・下限でクリップ。異常値は別表示にする。
大量 series 非表示の系列が多数あっても更新処理が走り、描画・操作が重くなる。 多数系列の監視画面でブラウザが固まる。 非表示系列は chart data から外す、表示系列数を制限、サーバ側で間引きを行う。
💡 設計時のチェックリスト:
  • msg.ui_update.chartOptions は変更時だけ送る（毎サンプル送らない）
  • chart ノード前で 必ず 入力データを検証し、series / x / y が欠けたデータを除外する
  • 長期稼働の監視画面では、ブラウザ更新・ページ切替・再接続時の再描画テストを必ず実施する
  • X 軸ラベルは内部値を一意にし、表示用フォーマットを別途指定する

💊 まず試したい応急処置：「Clear Chart History」ボタン

上記のような不具合（凡例に undefined が残る、過去データが壊れている、 複数系列の表示が崩れているなど）が発生した場合、 chart ノードの編集ダイアログ左上にある「Clear Chart History」ボタン を押すと、内部に蓄積された履歴データがクリアされて表示が正常に戻るケースがあります。 根本解決ではありませんが、配信中のダッシュボードを止めずに復旧したい場合に有効です。

▲ chart ノードの編集ダイアログ。左上の「Clear Chart History」ボタンで履歴をクリアできる。

💡 「Clear Chart History」の使いどころ:
  • 凡例に undefined が出てしまった後、入力データ側を修正したのに表示が戻らない
  • テスト中に投入した不正データが残っていて、本番データの表示を邪魔している
  • series 設定変更後に古い系列が残ってしまっている
  • 長時間稼働で動作が重くなってきた場合の暫定リセット

押下後は 「完了」ボタンを押してデプロイ することで反映されます。 クリア後にまた症状が再発する場合は、入力データ側のフィルタリングや chartOptions の送信頻度などを併せて見直してください。

最新の不具合情報・修正状況は FlowFuse Dashboard の GitHub Issues を参照してください。

✅ 8. まとめ

📊 データ可視化ウィジェットの使い分け

ウィジェット 最適な用途 入力データ形式
chart (Line) 時系列データの推移、トレンド分析 数値 + topic
chart (Bar) カテゴリ別比較、期間別集計 配列 / オブジェクト
chart (Pie) 構成比、割合の表示 配列 / オブジェクト
gauge (3/4, Half) 現在値の直感的表示、しきい値監視 数値
gauge (Tile) コンパクトな数値表示 数値
gauge (Battery, Tank) 残量、レベル表示 数値（%）
table 一覧表示、詳細データ確認 オブジェクト配列
progress 進捗状況、完了率 数値（%）

💡 重要ポイント

  • 時系列チャート: X-Axis Limitを設定して表示期間を制限（メモリ節約）
  • 複数系列: msg.topicでデータをグループ化
  • ゲージのセグメント: 閾値に応じた色分けで異常を視覚化
  • 動的プロパティ: msg.ui_updateで実行時に設定変更可能
  • チャートクリア: 空配列 [] を送信してリセット

🏭 10. 実務活用例

ケース1: 工場環境モニタリング

温度・湿度・気圧センサー → chart（時系列推移） → gauge（現在値表示） → 閾値超過時にnotification

ケース2: 生産ライン稼働状況

各設備の稼働率データ → chart（棒グラフで比較） → progress（全体稼働率） → table（設備別詳細）

ケース3: エネルギー管理ダッシュボード

電力・ガス・水道使用量 → chart（時間帯別消費量） → gauge（リアルタイム消費） → chart（円グラフで構成比）

ケース4: IoTセンサーネットワーク

複数センサーからのMQTTデータ → table（センサー一覧・状態） → chart（選択センサーの履歴） → gauge（バッテリー残量）
