## 7. Excel 対比の共通辞書

全ページで語彙を統一するための早見表。①で初出、他ページで参照する形。

| Excel | Node-RED | 備考 |
|-------|----------|------|
| セルの値 | `msg.payload` | 1 つの「入れ物」と中身 |
| 関数（`SUM`, `IF`） | Change / Switch / Function | 計算・判定 |
| マクロ全体 | 1 つのフロー（フロータブ）| 自動化の単位 |
| シート内セル参照 | フロー変数（flow context）| 同じタブ内で共有 |
| ブック全体で共有 | グローバル変数（global context）| 全タブで共有 |
| `Worksheet_Change` 等イベント | Inject / MQTT In / HTTP In など | 「何かが起きたら動く」|
| グラフ | Dashboard 2.0 の ui-chart / ui-gauge | 可視化 |
| フィルタ | Switch / Filter (RBE) | 条件で選別 |

## 8. 用語統一ルール

**設置場所**: ①のページ末尾（§9 第0章 用語辞典）にまとめて掲載。②〜⑥からは冒頭ナビで「用語がわからなくなったら → ①の用語辞典へ」とリンクで戻れる構造にする。

- 「フロー」= 1 つのタブ内のノードの繋がり全体
- 「タブ」= エディター上部に並ぶ作業エリアの単位
- 「ノード」= 処理を担う四角い箱（1つの機能）
- 「ワイヤー」= ノードとノードを繋ぐ線
- 「メッセージ」または「msg」= ワイヤーを流れるデータ
- 「ペイロード」または「msg.payload」= メッセージの主な中身
- 「トピック」または「msg.topic」= メッセージに付けるラベル
- 「トリガー」= フローを動かし始めるイベント
- 「デプロイ」= 編集したフローを実行状態に反映する操作（「保存」とは言わない）
- 「デバッグサイドバー」= エディター右側に表示されるデバッグ出力領域
- 「Dashboard」= 固有名詞（Dashboard 2.0）、小文字の「ダッシュボード」= 一般名詞（画面）として使い分ける
- 「コンテキスト」= フロー実行中にデータを覚えておく領域（詳細は variable-types Part2）
- 「ブローカー」= MQTT の中継サーバー（詳細は⑤と MQTT ガイド）
