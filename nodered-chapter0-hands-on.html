🌱 第0章③ 最初の10分ハンズオン

📖 このページの前提Node-RED がインストール済みで、ブラウザでエディターが開ける状態を想定しています（まだの方は Windowsインストール／Raspberry Piインストール）。①②は未読でも進められますが、読んでおくと理解がスムーズです。

🎯 得られるもの「イベントで動く」「データが流れる」「途中で加工できる」「画面に出せる」を 10 分で体験できます。手を動かすことで Node-RED の全体像が体に染みつきます。

📘 用語を忘れたら①の用語辞典に戻れます。

➡️ 次に読むページ④ 動かないときの進め方

🎯 このページのゴール

合計 10 分で、以下の 4 ステップを完走します。どのステップも単独でインポート可能な完成フロー JSON を用意しているので、途中で詰まっても再開できます。

  • ステップ1（2分）: Inject + Debug で「動いた！」を体験
  • ステップ2（2分）: 送るデータ（payload）を変えてみる
  • ステップ3（3分）: 途中に Change を挟んで値を加工する
  • ステップ4（3分）: ダッシュボードに結果を表示する

事前準備

  1. Node-RED が動いている PC で、ブラウザから http://localhost:1880 を開いてください（Raspberry Pi で動かしている場合は IP アドレスに差し替え）。
  2. 左側にノードのパレット、真ん中にフロー編集エリア、右側にサイドバーが見えていれば準備 OK です。
  3. 右上に赤い「デプロイ」ボタンがあることを確認してください。編集した内容はこのボタンを押さないと実行されません。

ステップ1：Hello をログに出す ⏱ 2分

1

Inject + Debug で最小のフローを作る

ゴール: Inject ボタンを押すと、デバッグサイドバーに時刻が表示される
  1. 左のパレットから Inject ノードを中央の編集エリアにドラッグ＆ドロップします。
  2. 同じように Debug ノードも配置します。
  3. Inject ノードの右側の小さな四角から、Debug ノードの左側の四角までワイヤーを繋げます（クリックしたまま引っ張る）。
  4. 右上の赤い「デプロイ」ボタンを押します。
  5. 右サイドバー上部の虫アイコン（🐛）をクリックして、デバッグサイドバーを開きます。
  6. Inject ノード左端の青いボタンをクリックします。
  7. デバッグサイドバーに数字（ミリ秒単位の時刻）が表示されれば成功です！
📸 スクリーンショット #2: Step1 完成フロー
撮影対象: Inject と Debug がワイヤーで繋がった最小フロー。デバッグサイドバーも開いた状態で、数字が 1 件表示されているところ。
推奨ファイル名: img/chapter0/handson-02-step1-flow.png
💡 何が起きたか
Inject ノードは「押されたら msg を発射する」イベント源。Debug ノードは「受け取った msg の中身をサイドバーに表示する」出力先。その 2 つをワイヤーで繋ぐことで、1 つのフローが完成しました。これが Node-RED のすべての基本形です。

🚧 動かないときはここを確認

  • デプロイボタンを押しましたか？ 編集しただけでは反映されません。
  • ワイヤーは本当に繋がっていますか？ Inject の右ポートから Debug の左ポートへ、線が引けているか目視確認。
  • デバッグサイドバーは開いていますか？ 右上の虫アイコンをクリック。

ステップ2：送るデータを変える ⏱ 2分

2

Inject の payload をいろいろ変えてみる

ゴール: 文字列・数値・JSON の違いを体感する
  1. ステップ1 の Inject ノードをダブルクリックして設定画面を開きます。
  2. msg.payload = ... の行の「日時」と書かれた部分を、プルダウンから「文字列」に変更します。
  3. 右の入力欄に こんにちは と入力して「完了」ボタンを押します。
  4. デプロイして、Inject ボタンをクリック。デバッグサイドバーに「こんにちは」と表示されるはずです。
  5. 同じ要領で、「数値」にして 25 を入れてみる。デプロイ、クリック、確認。
  6. 次は「JSON」にして {"temp":25,"unit":"℃"} を入れてみる。オブジェクトとして表示されます。
📸 スクリーンショット #3: Inject 設定画面
撮影対象: Inject ノードをダブルクリックした設定画面。「msg.payload」のプルダウンが開いて「文字列」が選ばれている様子。
推奨ファイル名: img/chapter0/handson-03-inject-payload-types.png
💡 何が起きたか
msg.payload の中身は、「型」によって扱いが変わります。文字列 "25"、数値 25、オブジェクト {...} はどれも違うものです。デバッグサイドバーでも色が違って表示されるので見比べてみてください。型の詳細は 変数ガイド Part 1 で扱います。

🚧 動かないときはここを確認

  • Inject の設定を変えたあと、必ずデプロイを押す（内容を変えただけでは実行中のフローに反映されない）。
  • JSON を入れる時に、中括弧やクォートが半角になっているか確認（全角だとエラー）。
  • 値を変えても何も変わらない場合は、ステップ1 の完成フロー JSON を読み込んで最初からやり直すのも早い。

ステップ3：途中で加工する ⏱ 3分

3

Change ノードで値を書き換える

ゴール: Inject → Change → Debug の 3 ノード構成を作り、途中で payload を変える
  1. Inject の payload を「文字列」 で こんにちは に設定しておきます。
  2. パレットから Change ノードを 1 つ、Inject と Debug の間にドラッグします。
  3. 既存のワイヤー（Inject → Debug）の真ん中に Change を落とすと、自動で割り込みます。もしうまくいかなければ、既存のワイヤーを削除して手動で繋ぎ直します。
  4. Change ノードをダブルクリックして設定画面を開きます。
  5. ルール部分を次のようにします。
    • プルダウン1: 「置換」（Change）
    • 対象: msg.payload
    • 検索する値: こんにちは（文字列）
    • 置換後の値: おはよう（文字列）
  6. 完了 → デプロイ → Inject クリック。デバッグサイドバーに おはよう と表示されれば成功です。
📸 スクリーンショット #4: Step3 完成フローと Change 設定
撮影対象: 3 ノード（Inject → Change → Debug）が繋がったフロー。可能であれば Change の設定画面を開いた状態も別ショットで。
推奨ファイル名: img/chapter0/handson-04-step3-flow.png
💡 何が起きたか
メッセージがフローを流れる途中で、Change ノードが「こんにちは」を見つけて「おはよう」に書き換えました。これがNode-RED の「処理」の正体です。ノードは受け取った msg に何かをして、次のノードへ送る、という動きを延々とやっています。

🚧 動かないときはここを確認

  • Change ノードのルールが「置換（Change）」になっているか（「値の代入（Set）」と間違えやすい）。
  • 検索文字列と Inject の payload が完全に一致しているか（半角・全角、スペースに注意）。
  • ワイヤーがInject → Change → Debug の順に繋がっているか。Change を迂回していないか。

ステップ4：ダッシュボードに出す ⏱ 3分

4

ui-text ノードで画面に表示する

ゴール: ブラウザに専用ダッシュボードを開き、加工後の文字列を表示する

事前準備: Dashboard 2.0 のインストール

ステップ4 では ui-text という画面表示用のノードを使います。これは Node-RED の標準では入っていない追加パッケージなので、先にインストールしておく必要があります。初めての方はここで 2〜3 分かかりますが、一度入れれば以降ずっと使えます。

✅ まず確認: 左のノードパレットを下にスクロールして、「dashboard 2」というカテゴリがあり、その中に ui-text が見えていればインストール済みです。この節はスキップして先に進めます。見当たらない場合は、下のインストール手順を実施してください。

インストール手順（3〜5 ステップ）

  1. Node-RED エディター右上のハンバーガーメニュー（≡ の形）をクリック。
  2. メニューの中から 「パレットの管理」 を選択。
  3. 開いたダイアログの 「ノードを追加」 タブに切り替え。
  4. 検索欄に @flowfuse/node-red-dashboard と入力（部分入力でも候補に出てきます）。
  5. 「@flowfuse/node-red-dashboard」（説明文に "Dashboard 2.0" とあるもの）の右の 「ノードを追加」ボタンをクリック。
  6. 確認ダイアログで「追加」をクリック。インストール完了まで 1 分ほど待ちます（バックグラウンドで npm install が走っています）。
  7. 完了すると、左パレットに 「dashboard 2」 カテゴリが追加され、ui-text などが選べるようになります。
⚠️ 最大の落とし穴: 旧版（Dashboard 1.0）と間違えない

検索結果には似た名前のパッケージが複数出てきます。必ず下の「◯」の方を選んでください。間違えて旧版を入れると、このガイドのフロー JSON は一切動きません。

判定 パッケージ名 正体
◯ これを入れる @flowfuse/node-red-dashboard Dashboard 2.0（現行、FlowFuse 社がメンテ）
✕ 入れない node-red-dashboard Dashboard 1.0（旧版、ノード種別が全く違う）

見分け方: パッケージ名の先頭に @flowfuse/ が付いているか。付いていれば 2.0、付いていなければ 1.0 です。

🚧 インストールで詰まったときの確認

  • 「パレットの管理」が見つからない: Node-RED のバージョンが古い可能性があります。Node-RED 3.x 以降を使用してください。
  • 検索しても出てこない: インターネット接続を確認。企業プロキシ配下の場合は管理者に確認が必要なことがあります。
  • インストールに失敗する: Node-RED ログ（起動したターミナル）にエラーが出ていないか確認。権限エラーなら ~/.node-red ディレクトリの所有者を確認。
  • インストール後もパレットに出ない: Node-RED を一度再起動してください。
  1. ステップ3 の Debug ノードは残したまま、ui-text ノードを追加で配置します。
  2. Change ノードの出力から2 本目のワイヤーを引いて、ui-text に繋ぎます（Change は 1 つの出力から複数のノードに送れます）。
  3. ui-text ノードをダブルクリックして設定画面を開きます。
  4. Group の右にある鉛筆アイコンをクリックして、新しいグループを作ります。
    • Group の名前: メイン（何でも OK）
    • Page の右の鉛筆アイコンで、新しいページも作成。
      • Page の名前: ページ1
      • UI の右の鉛筆アイコンで、ui-base も作成（名前はデフォルトのままで OK）
  5. Label には 挨拶、Value format には {{msg.payload}} と入力（初期値のまま）。
  6. 完了 → デプロイ。
  7. ブラウザで新しいタブを開き、http://localhost:1880/dashboard にアクセス。
  8. Node-RED エディターに戻り、Inject ボタンをクリック。
  9. ダッシュボード画面に おはよう と表示されれば成功です！🎉

④ 動かないときの進め方

プログラミング初心者にとって、動かない時間は「学びの時間」です。どう調べれば原因にたどり着けるか、その手順を具体化します。ここを覚えると、後の学習効率が圧倒的に変わります。

🏠
