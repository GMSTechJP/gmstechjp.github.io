ローカル/ラズパイ 演習環境ガイド

🖥️🍓 ローカル/ラズパイ向け 環境とは？

📖 このガイドの役目学習環境ガイドで「第2層（ローカル/ラズパイ）へ移ろう」となった方向けの実践セットアップです。TCP/UDPの自由な実験、MODBUS（シミュレータや実機）、ブロードキャストなど、共有クラウドでは難しかった演習を自分の環境で動かします。

🎯 前提Node-RED 自体のインストールは Windows ／ Raspberry Pi のインストールガイドで済ませておいてください。本ガイドはその「先」の演習準備です。

🧭 環境の考え方ローカルPCやラズパイはクラウドの上位互換（クラウドでできることは全部でき、＋自由なポートや実機）。詳しくは 学習環境ガイド。

0. 共通：ローカル/ラズパイで何が変わる？

自分専用のマシン（ローカルPCやラズパイ）でNode-REDを動かすと、共有クラウドの制約が外れます。

項目共有クラウドローカル/ラズパイ
ポート他の学習者と取り合い（固有番号が必要）自由（9000でも502でも好きに）
特権ポート（502等）開けない（非root）開ける（管理者/sudo）
ブロードキャスト基本届かない実LANで届く
物理ハード無しUSB機器（ローカル）／GPIO、センサー（ラズパイ）
自分のIPアドレスを把握しておく
別の機器（もう1台のPCやラズパイ）とTCP/UDP通信するには、相手のIPが要ります。確認コマンド：
Windows → ipconfig（IPv4アドレス）／ ラズパイ・Mac・Linux → hostname -I

1. Windows 編

手元のWindows PCは、MODBUSシミュレータ（GUI）やTCP/UDPの自由演習に向いた手軽な環境です。

1-1. MODBUS シミュレータ「ModRSsim2」で試す 🖥️ ローカル

ModRSsim2 は無料のWindows用Modbusスレーブシミュレータ（GUI）です。レジスタ値を画面で見ながら編集でき、初学者に最適です。

  1. 「ModRSsim2」を入手して起動する（Modbus TCP/シリアル両対応の擬似スレーブ）
  2. プロトコルを Modbus TCP、待受ポートを 10502（MODBUSガイドのサンプルと同じ番号にすると無編集で繋がります）に設定
  3. レジスタ表（Holding/Input など）に適当な値を入れておく
  4. Node-RED 側の Modbus-Client 設定ノード を 127.0.0.1、ポート 10502 に向ける（サンプルの既定値）
  5. Modbus-Read でポーリングし、Debugに値が出れば成功
ポートは「シミュレータ」と「フロー」で揃えるのが鉄則。本サイトのサンプルは 10502 固定なので、シミュレータも 10502 にすればそのまま繋がります。標準の 502 を使いたい場合は、シミュレータとフロー側（Modbus-Client）の両方を 502 に変更してください（ローカルは管理者権限があるので 502 も開けます）。
CLI派なら diagslave
GUIが不要なら、MODBUSガイドで紹介した diagslave -m tcp -p 10502 でもスレーブを立てられます（Windows/Mac/Linux共通）。

1-2. USBシリアルで MODBUS RTU（実機/変換器） 🖥️ ローカル

USB-RS485 変換器をPCに挿すと、Windowsでは COMポート（例 COM4）として認識されます。Modbus-Client 設定ノードを Serial に切り替え、そのCOMポートとボーレート（機器に合わせる、例 9600）を指定すれば、RTUの実機と通信できます。

1-3. TCP/UDP を自由に試す 🖥️ ローカル

ローカルではポート番号を気にせずサンプルのまま（9000, 8000 等）使えます。同じLANのスマホや別PCと送受信する実験も可能です。

Windowsファイアウォールに注意
他の機器から自分のNode-RED（TCP/UDPの待受やHTTP）へ接続できないときは、Windows Defender ファイアウォールが受信をブロックしている可能性大。初回接続時に出る許可ダイアログで「許可」するか、該当ポートの受信を許可してください。なお、同じPC内（127.0.0.1）の通信はブロックされません。

2. Raspberry Pi 編

ラズパイは、TCP/UDPの実LAN実験に加え、シリアル（RS-485）、GPIO、センサーといった物理ハードまで扱える本命環境です。

2-1. 同じLANで TCP/UDP 実験 🍓 🖥️

ラズパイともう1台（PCや別のPi）を同じLANにつなげば、2台の間で実際のTCP/UDP通信を体験できます。片方を「Listen on（サーバー）」、もう片方を「Connect to（クライアント）」にして、IP同士でつなぎます。共有クラウドではできなかった「機器間通信」の本番です。

2-2. ブロードキャスト実験 🍓 🖥️

実LANなら、UDPのブロードキャスト（255.255.255.255 など）が機能します。機器発見や一斉通知の演習は、ここで試してください（クラウドでは届きません）。

2-3. RS-485 で MODBUS RTU 実機 🍓

ラズパイに RS-485 HAT や USB-RS485 変換器をつなぐと、PLC、インバータ、電力計などの実機とMODBUS RTUで通信できます。

  • USB変換器 → /dev/ttyUSB0 として認識
  • GPIO上のUART/HAT → /dev/serial0（raspi-config でシリアルを有効化し、シリアルコンソールは無効化）
  • Modbus-Client 設定ノードを Serial にし、上記ポート、ボーレート、パリティを機器に合わせる
シリアルポートの権限（dialout）
Linux/ラズパイでシリアルポートを使うには、ユーザーが dialout グループに属している必要があります。
sudo usermod -a -G dialout $USER # 反映には再ログイン（または再起動）が必要
権限不足だと「ポートを開けない」エラーになります。

2-4. GPIOとセンサー（Sense HAT 等） 🍓

温度、湿度、モーションなどの物理センサーや、LEDやリレーの制御はラズパイならでは。Pi Sense HAT ガイドで具体的な扱い方を解説しています。

4. まとめ

  • ローカルPCやラズパイはポート自由、特権ポート可、ブロードキャスト可、実機接続可。クラウドの制約が外れる。
  • Windows：ModRSsim2でMODBUSを手軽に、USB-RS485でRTU実機、TCP/UDPは自由に。ファイアウォールに注意。
  • ラズパイ：実LANでのTCP/UDPやブロードキャスト、RS-485でRTU実機、GPIOやセンサー。シリアルは dialout 権限。
  • どの演習をどこでやるかは 学習環境ガイド の環境マップで確認。
