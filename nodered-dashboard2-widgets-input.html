🎯 1. 入力系ウィジェットの概要

🤔 入力系ウィジェットとは？

入力系ウィジェットは、ユーザーからの入力を受け付けてNode-REDフローに送信するためのウィジェットです。 日常生活で例えると、リモコンや操作パネルのようなものです。

🎮 ゲームコントローラーに例えると:

  • 🔘 button = Aボタン・Bボタン（押すとアクション実行）
  • 🔛 switch = 電源スイッチ（ON/OFFの切り替え）
  • 🎚️ slider = 音量つまみ（連続値の調整）
  • 📋 dropdown = モード選択メニュー（リストから選択）
  • ⌨️ text-input = 名前入力欄（文字の入力）
  • 📝 form = 設定画面（複数項目の一括入力）
  • 🔘 radio-group = 難易度選択（排他的な選択）
  • 🎛️ button-group = アイテム選択ボタン群

📊 入力系ウィジェット一覧

ウィジェット 用途 出力値の型 入力受付
button クリックでアクション実行 設定値（任意） 状態更新可
switch ON/OFF切り替え boolean / 設定値 ✅
slider 数値の範囲入力 number ✅
dropdown リストから選択 選択した値 ✅
text-input テキスト入力 string / number ✅
form 複数フィールド入力 object ✅
radio-group 排他的選択 選択した値 ✅
button-group ボタン群から選択 選択した値 ✅

🔘 9. radio-group（ラジオボタン）

入出力 radio-group ウィジェット

複数の選択肢から1つだけを選ぶラジオボタングループです。選択肢が少ない場合に視認性が高く便利です。

📋 主要プロパティ

radio-group ウィジェットは ja ロケールがなく、エディター表示は英語（「名前」は共通項目なので日本語）。英語項目は「英語ラベル（日本語訳）」で示す。表はエディターの上から下への並びに沿う。

Label（ラベル） グループの横に表示する文字列。 （空）
Class（クラス） 任意の CSS クラス名。 （空）

🎓 12. まとめ

入力系ウィジェットの重要ポイント

  • ✅ button: クリックで任意の値を送信、アイコン・色のカスタマイズ可能
  • ✅ switch: ON/OFF切り替え、カスタム値・アイコン対応
  • ✅ slider: 範囲内の数値入力、操作中/完了時の出力選択可能
  • ✅ dropdown: 単一/複数選択対応、動的オプション変更可能
  • ✅ text-input: 様々な入力モード（text/number/password/email/date等）
  • ✅ form: 複数フィールドをまとめて送信
  • ✅ msg.payloadで外部から値を設定可能（双方向バインディング）
  • ✅ msg.ui_updateで動的にプロパティを変更可能

⚠️ よくある間違い

  • Pass Throughの誤解: 入力メッセージをそのまま出力に流す機能（状態更新とは別）
  • Decoupleの誤解: UIからの操作時のみ出力する機能（入力での更新は出力しない）
  • ui-notificationの参照: groupではなくuiプロパティでui-baseを参照
  • ui-formの出力: 各フィールドのkeyがオブジェクトのキーになる
  • スライダーのOutput設定: allは操作中常時、endは操作完了時のみ

🔧 13. トラブルシューティング

よくある問題と解決方法

問題 原因 解決方法
ウィジェットが表示されない Groupが未設定 ウィジェットにGroupを設定する
値が送信されない ワイヤー未接続 出力ポートにワイヤーを接続する
スイッチの状態が更新されない 送信値と設定値の型不一致 On/Off Valueの型を確認（bool/str/num）
ドロップダウンの選択肢が変わらない msg.optionsの形式エラー 配列形式で{label, value}オブジェクトを送信
フォームの初期値が設定されない keyの不一致 msg.payloadのキーとフィールドのkeyを一致させる
通知が表示されない ui参照エラー ui-notificationのuiプロパティでui-baseを参照
