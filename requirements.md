# 追加ガイド 要件定義（バイナリ処理・ファイル転送・PLC）

今後追加・拡充するガイドの**要件の正本**。詳細な調査・設計・選定根拠は
[docs/roadmap.md](docs/roadmap.md) を参照。サイト全体の開発規約は
[CLAUDE.md](CLAUDE.md)、文章・設定項目の基準は
[docs/style-guide.md](docs/style-guide.md) を正とする。

作成日：2026-06-29
ステータス：方針確定・未着手

---

## 1. 目的

バイナリデータ処理（Buffer・エンディアン）とファイル転送（FTP/SFTP）の理解を
補う 4 テーマのガイドを追加・拡充する。3 テーマ（A・B-1・C）は「バイナリデータを
どう読み書きするか」で一本につながる学習動線とする。

## 2. 対象スコープ（4 テーマ）

| ID | テーマ | 成果物 | 方針 |
| --- | --- | --- | --- |
| A | バイナリ/テキストと Buffer | 既存 `variable-types`（part2 軸）へ**追記** | 新規ファイルは作らない |
| B-1 | buffer-parser ノード | 新規ガイド | サードパーティ |
| B-2 | FTP / SFTP | 新規ガイド | server / client 両対応 |
| C | PLC レジスタ（エンディアン・Modbus 注意点） | **新規独立ガイド** | `modbus` 等から参照先とする |

## 3. 採用パッケージ（確定／npm 実調査に基づく）

| 用途 | パッケージ | 補足 |
| --- | --- | --- |
| B-1 変換 | `node-red-contrib-buffer-parser` | ビッグ/リトルエンディアン対応 |
| B-2 FTP クライアント | `node-red-contrib-ftp` | DL・star 最多 |
| B-2 FTP サーバー | `node-red-contrib-ftp-server` | DL・star 最多 |
| B-2 SFTP クライアント | `node-red-contrib-ssh-sftp`（第一）／ `node-red-contrib-sftp`（実績） | 前者は近年更新・ssh2-sftp-client ベース |
| B-2 SFTP サーバー | `node-red-contrib-sftp-server` | 選択肢が乏しい。OS の SSH デーモン代替も補足 |

着手時に各パッケージの最新版・DL 数・更新状況を再確認する。

## 4. 着手順

**A → B-1 → C → B-2**（B-2 に SFTP を含む）。

## 5. 品質要件（必須）

- 設定項目名はエディターの実表示に完全一致。ja locale が無い場合は
  「**English Label**（日本語訳）」形式。
- サンプルフローの JSON は `python3 -m json.tool` でパース成功すること。
- サードパーティガイドは「インストール方法・GitHub リンク・バージョン・免責」を明記。
- 設定項目の節はエージェントへ丸投げせず、1 ノードずつソースと挙動を確認して書く。
- フローティングホームボタン・前後リンク・コピーボタンを備える。
- 既存ガイドへの橋渡しリンクは実在ファイル名を指す。

## 6. 制約・注意

- 公開リポジトリ。機密情報・API キーを含めない。
- 平文 FTP は認証情報が平文で流れる。LAN 内・検証用に限定し、インターネット越しでは
  SFTP/FTPS を勧める注記を必須とする。
- FTP/SFTP サーバー系パッケージは更新が停滞している。トレードオフを本文に明記する。
- C は `modbus` ガイドと重複させない。アドレス体系（0 始まり・4xxxx）は既出のため、
  C は「レジスタ値のバイト解釈」に特化する。

## 7. 受け入れ基準

- 4 テーマの成果物が第 5 章の品質要件をすべて満たす。
- 第 2〜4 章の方針（追記 / 新規独立 / 採用パッケージ / 着手順）と一致する。
- 内容が [docs/roadmap.md](docs/roadmap.md) の設計と矛盾しない。
