💡 Node-REDとは？

Node-REDは、ブラウザ上で動作するビジュアルプログラミングツールです。 「ノード」と呼ばれる部品を線でつなぐだけで、複雑なプログラムを簡単に作成できます。

🖱️
ドラッグ&ドロップ

マウス操作だけでプログラミング。コードを書く必要はほとんどありません。

🔌
IoTに最適

センサーやデバイスとの連携が簡単。Raspberry Piとの相性も抜群です。

🌐
API連携

WebサービスやクラウドAPIとの接続も簡単に実現できます。

⚡
リアルタイム

変更したらすぐにデプロイ。動作確認がスピーディーに行えます。

🖥️ エディタ画面の構成

Node-REDのエディタは主に4つのエリアで構成されています。実際の画面を見てみましょう。

画面構成の詳細説明

① パレット（左側）

使用可能なノードが一覧表示されます。

  • 検索バー： ノードを名前で検索
  • 共通カテゴリ： inject（入力）、debug（デバッグ）など
  • 機能カテゴリ： function（関数）、switch（分岐）など
  • 色分け： ノードの種類ごとに色が違う

② ワークスペース（中央）

フローを作成するメインの作業エリアです。

  • グリッド背景： ノードを整列しやすい
  • タブ表示： 複数のフローを切り替え可能
  • ドラッグ&ドロップ： パレットからノードを配置
  • 接続： ノード間を線でつなぐ

③ デバッグパネル（右側）

debugノードの出力を表示するエリアです。

  • リアルタイム表示： データの流れを確認
  • フィルター機能： 特定のフローのみ表示
  • メッセージ詳細： JSONデータの構造を確認
  • デバッグ支援： 問題箇所を特定しやすい

④ ツールバー（上部）

操作ボタンとフロー管理エリアです。

  • デプロイボタン： 変更を反映（最重要！）
  • メニュー： 設定、インポート/エクスポート
  • フロー名： 現在編集中のフロー
  • 情報パネル： ノードの説明を表示

💡 基本操作ガイド

🖱️ ノードの配置：
パレットのノードをワークスペースにドラッグ&ドロップ
🔗 ノードの接続：
ノードの出力ポート（右）から入力ポート（左）へドラッグ
⚙️ 設定の編集：
ノードをダブルクリックして設定画面を開く
🗑️ 削除：
ノードまたは接続線を選択してDeleteキーを押す
🚀 デプロイ：
右上の「デプロイ」ボタンをクリック（変更を反映）
🐛 デバッグ：
debugノードを接続して右パネルで出力を確認
⚠️ 重要なポイント： フローを編集したら必ず「デプロイ」ボタンを押してください。 デプロイしないと変更が反映されません。デプロイボタンが赤く表示されている時は未保存の変更があります。

🔗 ノードとフローの基本

ノードは処理の最小単位で、フローはノードをつないだ処理の流れです。

基本的なフローの例

📥
Inject
入力ノード
→
🔧
Function
処理ノード
→
🐛
Debug
出力ノード

ノードの種類

入力ノード（Input）
データを取得・受信します
例: inject, mqtt in, http in
処理ノード（Processing）
データを加工・変換します
例: function, change, switch
出力ノード（Output）
データを送信・保存します
例: debug, mqtt out, http response

💬 データの流れ（メッセージング）

Node-REDでは、ノード間でmsgオブジェクトとしてデータを受け渡します。

メッセージオブジェクトの構造

{
  "payload": "こんにちは", // メインのデータ
  "topic": "greeting", // トピック
  "timestamp": 1234567890 // タイムスタンプ
}

payload: 一番重要なデータが入る場所
topic: データの種類や分類
その他: 自由にプロパティを追加できます

メッセージは「バケツリレー」のように次々と渡されます

Inject
msg.payload = 25
→
Function
msg.payload × 2
→
Debug
表示: 50

✅ まとめ

Node-REDの3つの強み：

  • ✨ 視覚的で直感的 - コードを書かずに開発できる
  • 🚀 すぐに試せる - デプロイが瞬時、デバッグも簡単
  • 🔌 豊富な連携 - IoTデバイスからクラウドまで幅広く対応

🎯 次のステップ：

  1. Node-REDをインストールして起動してみる
  2. inject → debug の簡単なフローを作ってみる
  3. 公式ドキュメントのチュートリアルに挑戦
  4. 自分のIoTプロジェクトに応用してみる
