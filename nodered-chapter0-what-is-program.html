1. なぜ「プログラムとは？」から始めるのか

Node-RED はビジュアルで分かりやすい反面、実は多くの初心者が「ノードの使い方は分かるのに、なぜ思い通りに動かないのか」でつまずきます。その多くは、ノードそのものではなく、プログラム全般に共通する考え方が土台にないことが原因です。

このページでは、あえて Node-RED の話をほとんどせず、Excel の経験だけを頼りに「プログラムとは何をするものか」を整理します。ここを押さえると、以降のページが一気に見通しよくなります。

2. プログラムとは「コンピュータに渡すレシピ」

プログラムを一言で言えば、コンピュータに「こうやって動いてね」と渡す手順書です。料理のレシピと同じで、材料（データ）を用意し、手順（処理）を書き、できあがり（結果）を得る、という構造をしています。

🍳 料理のレシピ

材料を準備 → 手順に沿って調理 → 完成品
材料の順番を変えたり、手順を飛ばすと結果が変わる

🥤 自動販売機

お金を入れる → ボタンを押す → 商品とお釣りが出る
お金が足りなければ何も起きない、という「条件」もルール

📊 Excel のマクロ

データ範囲を選ぶ → マクロを実行 → 集計結果が表示
書いたステップを順番に、正確に実行するだけ

💡 コンピュータの性質
コンピュータは速く・正確に・疲れずに動きますが、融通は利きません。曖昧な指示や「だいたいこんな感じ」は通じず、書いた通りにしか動きません。プログラミングとは、この「融通の利かない相手」に正確な手順を渡す作業です。

3. コンピュータは 3 つのことしかできない

どんなに複雑に見えるプログラムも、本質的には次の 3 つの組み合わせでできています。この 3 つを覚えておけば、Node-RED でも Python でも JavaScript でも、本質は同じだと分かります。

📦

① 記憶する

値を覚えておく。
（例: 温度センサーの値を「25」として保持する）

🧮

② 計算・判断する

足し算・引き算、大小比較、条件判定。
（例: 25 が 30 より大きいか？）

🔌

③ 入出力する

外の世界から受け取り、外の世界に返す。
（例: センサーから読む、画面に出す、メールを送る）

📊 Excel で言うと

  • 記憶する = セルに値を入れる
  • 計算・判断する = 関数（=SUM(A1:A10)、=IF(A1>30, "警告", "正常")）
  • 入出力する = データ取り込み（CSV インポート等）、グラフや表で見せる
Node-RED も結局この 3 つをやっているだけです。見た目が違うだけで、本質は Excel と同じです。

4. すべてのプログラムは「入力 → 処理 → 出力」

もう一段シンプルにすると、プログラムは必ず次の 3 段階で動きます。

📥 入力
何かを受け取る ➜ ⚙️ 処理
計算・判定する ➜ 📤 出力
結果を出す

この 3 段階が、プログラムの一番大きな骨格です。Excel の関数も、BMI 計算も、Node-RED のフローも、みんなこの型に収まります。

具体例: BMI 計算

段階 Excel の場合 Node-RED の場合
入力 A1 セルに身長(m)、B1 セルに体重(kg) を入力 Inject ノードで身長と体重の値を送信
処理 C1 セルに =B1/(A1*A1) と書く Change ノードで BMI を計算する式を書く
出力 C1 の値を見る、グラフ化する Debug ノードで表示、ダッシュボードに出す
🔍 ここでの学び
Node-RED の 1 本のフローは、「入力ノード → 処理ノード → 出力ノード」を線（ワイヤー）で繋いだ形になります。これは Excel の「入力セル → 計算式 → 結果セル」と同じ発想です。

6. プログラムを貫く 3 つの流れ

「コンピュータは 3 つのことしかできない」と同じくらい大事な原則が、処理の進み方にも 3 種類しかないということです。

① 順次実行

上から下へ、前から後ろへ、書いた順番通りに実行する基本動作です。

Excel の例: マクロの記述を 1 行目から順に実行する。

日常の例: お米を研ぐ → 水を入れる → 炊飯ボタンを押す。順番を逆にすると意味が通らない。

② 条件分岐

「もし〜ならA、そうでなければB」のように、状況で道を分ける動作です。

Excel の例: =IF(A1>30, "警告", "正常")

日常の例: 雨なら傘を持つ、晴れなら帽子をかぶる。

③ 繰り返し

同じ処理を何回も繰り返す動作です。

Excel の例: 1000 行の表を全行チェックしていくマクロ。

日常の例: 洗濯物を 1 枚ずつ畳む（最後の 1 枚になるまで繰り返す）。

🎯 覚えておくポイント
どんな言語・どんなツールのプログラムも、順次・分岐・繰り返しの 3 つの組み合わせで書かれています。複雑に見えるコードも、この 3 種類の入れ子に過ぎません。

7. Node-RED ではこう表現する

3 つの流れを Node-RED で表すと次のようになります。見た目は違っても、やっていることは他の言語と同じです。

流れの種類 Excel・一般言語 Node-RED
順次実行 上から書いた順に実行 ノードをワイヤーで繋いだ順に msg が流れる
条件分岐 IF 関数、if 文 Switch ノード（条件で出力ポートを選ぶ）
繰り返し ループ処理 Split ノードで配列を分解、Inject ノードの「繰り返し」機能、Trigger ノード など
⚠️ 重要な違い
Node-RED は「上から順に実行」だけではありません。ボタンが押されたときや、定期タイマーが発火したときなど、何かのきっかけで動き始める仕組みが特徴です。これを「イベント駆動」と呼びますが、詳しくは次のページ ②イベント駆動とは？で解説します。まずは「上から順」だけではないことを覚えておいてください。

8. データには「型」がある（予告）

プログラムが扱うデータには種類（型）があります。Excel で言う「数値」「文字列」「日付」の区別と同じです。

  • 数値（1、25.4）
  • 文字列（"こんにちは"、"温度: 25度"）
  • 真偽値（true / false）
  • 配列（複数の値のリスト）
  • オブジェクト（複数の情報をまとめたもの）

型が違うと計算結果も変わります。たとえば文字列の "1" と数値の 1 は別物で、計算するとおかしな結果になることがあります。これは④のデバッグで繰り返し出てくる話題です。

9. 第0章 用語辞典

第0章全体で使う用語を一覧にまとめました。他のページで「この言葉何だっけ？」となったら、ここに戻ってきてください。

📖 用語一覧

ノード
Node-RED のエディター上で四角い箱として表示される、ひとつひとつの機能の単位。
例: 入力を担当する「Inject ノード」、値を加工する「Change ノード」、結果を表示する「Debug ノード」など。
タブ
Node-RED エディターの上部に並ぶ、作業エリアの区切り。1 つのタブ ＝ 1 つのまとまった仕事、と考えると分かりやすい。
フロー
1 つのタブ内にある、ノードとワイヤーで作られた処理の流れ全体。Excel で言えば「1 つのマクロ」にあたる。
ワイヤー
ノードとノードを繋ぐ線。ワイヤーを通じてデータが流れる。
メッセージ（msg）
ワイヤーを流れる 1 つ 1 つのデータの塊。郵便物や宅配便の「箱」のようなイメージ。中に複数の情報を入れて運ぶ。
ペイロード（msg.payload）
メッセージの中に入っている「主な中身」。温度センサーの値、送信する文字列、計算結果など。
Excel で言えば「セルの値」に最も近い概念。
トピック（msg.topic）
メッセージに付けるラベル。どこから来たメッセージか、何についてのメッセージかを示す目印。
トリガー
フローを動かし始める「きっかけ」となるイベント。ボタン押下、一定時刻の到来、メッセージ受信など。
デプロイ
エディター右上にある赤いボタンで、編集したフローを実際に動く状態に反映する操作。「保存」とは違い、これを押さないと変更が反映されない。
デバッグサイドバー
エディター右側に表示される、Debug ノードからの出力が一覧表示される領域。動作確認に必須の場所。
Dashboard（ダッシュボード）
ブラウザで見える可視化画面。グラフ、ゲージ、ボタンなどを配置してユーザーと対話できる。正式名称は Dashboard 2.0。
コンテキスト（context）
メッセージが流れ終わった後も、値を覚えておける記憶領域。Excel で言う「シート全体で参照できる値」に近い。詳細は 変数ガイド Part 2。
ブローカー（MQTT ブローカー）
MQTT 通信の中継サーバー。送信側と受信側の間に立って、メッセージを振り分ける役目。詳細は⑤IoT全体像と MQTT ノードガイド。

10. まとめと次のページ

このページで押さえたかったことは次の 4 つです。

  1. プログラムは コンピュータに渡すレシピ。書いた通りにしか動かない。
  2. コンピュータは 記憶・計算/判断・入出力 の 3 つを組み合わせているだけ。
  3. どんなプログラムも骨格は 入力 → 処理 → 出力。
  4. 処理の進み方には 順次・分岐・繰り返し の 3 種類しかない。

この土台ができると、Node-RED の画面に並ぶ様々なノードも「結局どれかの流れを表しているだけだ」と見えてきます。
