📖 1. 概要

Comment ノードは、フロー内にコメントやメモを残すためのノードです。プログラムのソースコードにコメントを書くのと同じように、フローの説明、注意事項、TODOなどを記録できます。

📚 本のしおりや付箋に例えると：Comment ノードは本に貼る付箋のようなものです。本文（フローの処理）には影響を与えませんが、読む人に重要な情報を伝えます。

📝 コメント （他のノードと接続しない独立したノード）

💡 ポイント:

  • Comment ノードはメッセージを受け取らず、送信もしない
  • フローの実行には一切影響を与えない
  • ドキュメントとしての役割を果たす

📝 5. 演習問題

演習1: 基本的なコメント追加 初級

📋 課題: Inject → Debug のフローを作成し、フローの説明コメントを追加してください。

✅ 成功の条件:

  • フロー上にCommentノードが配置されており、ノードのタイトルがフローエディタ上で視認できる
  • CommentノードをダブルクリックしてInfoフィールドにフローの説明が入力されている
  • CommentノードはInjectやDebugノードとワイヤーで接続されていない（独立して配置）
  • Injectボタンをクリックするとデバッグパネルに正常に出力される（Commentノードがフロー実行に影響していない）
💡 ヒント

フローの左上にCommentノードを配置すると見やすくなります。

演習2: セクション分けコメント 初級

📋 課題: 複数のセクションを持つフローを作成し、各セクションにコメントを追加してください。

✅ 成功の条件:

  • 「入力」「処理」「出力」など複数のセクションを表すCommentノードがそれぞれ対応する処理ノードの近くに配置されている
  • 各Commentノードのタイトルを見るだけでそのセクションの役割がわかる
  • フロー全体を俯瞰したとき、処理の流れが視覚的に把握できる構成になっている
  • フローは正常に実行できる（Commentノードの追加が実行に影響していない）
💡 ヒント

「━━━ セクション名 ━━━」のような区切り線を使うと視認性が向上します。

演習3: Markdown活用 中級

📋 課題: Markdownを使って、構造化されたドキュメントをCommentノードに作成してください。

✅ 成功の条件:

  • CommentノードのInfoフィールドに # や ## を使った見出しが含まれている
  • 箇条書き（-）または番号付きリスト（1.）が使われている
  • Node-REDエディタのInfoタブでCommentノードを選択したとき、Markdownがレンダリングされて見出しやリストとして表示される
  • システム仕様や設計意図が構造化された形で記述されている
💡 ヒント

見出し、箇条書き、番号付きリストを組み合わせて使いましょう。

演習4: プロジェクト管理 中級

📋 課題: TODO、バージョン履歴、警告を含むコメント体系を作成してください。

✅ 成功の条件:

  • 4種類以上の役割を持つCommentノード（概要・TODO・警告・変更履歴など）がフロー上に配置されている

✅ 6. まとめ

🎯 重要ポイント:

  • Comment ノードはフローの実行に影響を与えない
  • Info フィールドは Markdown 記法に対応
  • 絵文字で視認性を向上できる
  • フローの保守性向上に不可欠

⚠️ 注意事項:

  • コメントは定期的に更新し、古い情報を残さない
  • 過剰なコメントはかえって可読性を下げる
