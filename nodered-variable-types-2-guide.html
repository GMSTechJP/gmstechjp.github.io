📖 1. なぜcontext系変数が必要か？

msgオブジェクトは「ノードからノードへ流れる荷物」のようなもので、1本のフローの中でしか値を共有できません。しかし実際にアプリを作ると、msgだけでは解決できない場面に必ずぶつかります。

🤔 msgだけでは困る3つの典型例

🔢 場面A: 前回の値と比べたい

温度センサーが値を送ってきたとき、「前回より上昇したか下降したか」を判定したい。でも msg には今回の値しか入っていない。前回値をどこかに覚えておく必要がある。

🔀 場面B: 複数の入力経路を合わせたい

センサーAとセンサーBがバラバラのタイミングで別々のInjectから値を送ってくる。両方の最新値を使って合計や平均を計算したいが、msg.payload だけでは片方しか伝わらない。

🌍 場面C: 全タブで同じ設定を使いたい

APIキー、警告閾値、デバイス名などの設定値を複数のタブから参照したい。各フローに同じ値を書くとメンテナンスが大変。

💡 context系変数の役割：

これら3つの場面で必要になるのが「フローを超えて値を覚えておく仕組み」です。それが context / flow / global の3つのスコープを持つ context系変数です。「どこまでの範囲で共有したいか」によって、使い分けます。

🌐 2. context / flow / global の3つのスコープ

context系変数は、宣言の仕方（APIの呼び分け）によってスコープ（共有範囲）が変わります。どの範囲まで値を共有したいかによって使い分けましょう。

context（ノード）

範囲：定義されたノード内のみ

有効期限：Node-REDを再起動するまで

宣言方法：context.set() / context.get()

context.set('lastValue', 25); let val = context.get('lastValue');
📝 例：
特定のFunctionノード内でのみ使用する前回値や状態を保存

flow（フロー）

範囲：同じフロー（タブ）内のすべてのノード

有効期限：Node-REDを再起動するまで

宣言方法：flow.set() / flow.get()

flow.set('counter', 0); let count = flow.get('counter');
📝 例：
同じフロー内の複数ノードで共有するカウンター値や設定値

global（グローバル）

範囲：すべてのフロー（全タブ）

有効期限：Node-REDを再起動するまで

宣言方法：global.set() / global.get()

global.set('apiKey', 'xxx'); let key = global.get('apiKey');
📝 例：
全フローで共有するAPIキーやユーザー設定
💡 使い分けの3原則：
  • context: そのノードだけで覚えておきたいデータ（前回値、内部状態）
  • flow: 同じタブ内の複数ノードで共有したいデータ（カウンター、中継値）
  • global: すべてのタブで共有したいデータ（設定値、APIキー）— 使いすぎ注意
⚠️ 共通の注意点：
  • いずれも値はメモリ上にのみ保持される。Node-REDを再起動すると消える（永続化するには設定が別途必要）
  • global を乱用すると「どこから変更されているか」が追いにくくなる。本当に必要な時だけ使う

🔧 3. context系変数の使い方

context系変数は、ChangeノードのGUI操作と Functionノード内のJavaScript のどちらでも読み書きできます。目的と好みに応じて使い分けましょう。

🎨 Changeノードで読み書きする（JavaScript不要）

Changeノードの「値の代入」ルールで、対象や値のタイプとして flow / global を選べます。JavaScriptを書かなくてもGUI操作だけでcontext系変数を操作できます。

やりたいこと Changeノードの設定
msg.payload の値を flow.x に保存 ルール: Set → flow.x を msg.payload に設定
global.y の値を msg.payload に読み込む ルール: Set → msg.payload を global.y に設定
flow.counter に固定値 0 を入れる ルール: Set → flow.counter を 数値 0 に設定
flow.counter を +1 する ルール: Set → flow.counter を JSONata式 ($flowContext("counter") ? $flowContext("counter") : 0) + 1
📌 JSONata関数で context系変数を読む：
  • $flowContext("キー名") : flow変数の値を取得
  • $globalContext("キー名") : global変数の値を取得
  • これらは Changeノードの JSONata式フィールドや、Switchノード、Templateノードなどで使える

💻 Functionノードで読み書きする（JavaScript）

Functionノード内では、context / flow / global の3つのオブジェクトが使えます。条件分岐や計算ロジックが複雑な場合は、こちらのほうが柔軟です。

操作 API 例
値を保存 xxx.set(キー, 値) flow.set('counter', 5)
値を取得 xxx.get(キー) let n = flow.get('counter')
未設定時のデフォルト xxx.get(キー) || 既定値 let n = flow.get('counter') || 0
値を削除 xxx.set(キー, undefined) flow.set('counter', undefined)
// Functionノード内のコード例 // 現在の flow.counter を取得し、+1 して保存 let count = flow.get('counter') || 0; count += 1; flow.set('counter', count); // 取得した値を msg.payload に入れて次のノードに渡す msg.payload = count; return msg;
💡 どちらを使うか迷ったら：
  • 単純な保存・参照・代入だけ → Changeノード（コーディング不要）
  • 条件分岐、ループ、配列操作 → Functionノード（JavaScriptの柔軟性）
  • どちらで書いた変数でも、互いに読み書きできる（相互運用性あり）

🅰️ パターンA: Functionノードを使う方法

カウント → カウント+1 → 表示
リセット → flow.counter=0 → 表示

Functionノードのコード：

🅱️ パターンB: Changeノードだけで実現する方法（JavaScript不要）

Functionノードを使わず、Changeノード1つで 同じ処理を実現できます。JSONata式を使って「flow.counter を取り出して +1 した値を、再び flow.counter に保存」します。JavaScriptを書かなくてよいので、プログラミング初心者に優しい方法です。

フロー構成：

カウント → +1して保存 → 表示
リセット → flow.counter=0 → 表示

Changeノード「+1して保存」の設定（2ルール）：

ルール1: flow.counter を JSONata式 に設定 式: ($flowContext("counter") ? $flowContext("counter") : 0) + 1 （解説: 現在値が存在すればその値、無ければ0 として +1 し、flow.counter に書き戻す） ルール2: msg.payload を JSONata式 に設定 式: "カウント: " & $flowContext("counter") & "回" （解説: ルール1で更新済みの flow.counter を読んで文字列連結）
📌 Changeノード版の仕組み：
  • ルールは上から順に実行される
  • ルール1で flow.counter を更新 → ルール2で更新後の値を使える
  • $flowContext("counter") はChangeノード内でも使える JSONata関数
  • & は JSONata の文字列連結演算子（JavaScriptの + に相当）

💡 例2: global変数で「設定値」を全フローから参照する

よくある場面： 温度の警告閾値、デバイス名、APIエンドポイントなど、複数のタブ（フロー）から同じ設定値を参照したいケース。global変数は全タブから参照できるので、このような「設定値の共有」に最適です。

フロー構成：

設定登録 → global.threshold=30 → 登録確認
温度=35 → 判定 → 結果表示

ポイント：

  • 最初に「設定登録」Injectを押して global.threshold = 30 を保存
  • 次に「温度=35」Injectを押すと、Changeノードが global.threshold を取り出して比較
  • JSONata式で、現在温度・閾値・超過判定を1つのJSONにまとめて出力
  • 実運用では「設定登録」と「温度判定」は別タブに配置しても同じように動く

判定Changeノードの設定（2ルール）：

ルール1: msg.threshold を global.threshold に設定（値の代入） ルール2: msg.payload を JSONata に設定 式: { "現在温度": payload, "閾値": $globalContext("threshold"), "警告": payload > $globalContext("threshold") }

💡 例3: context変数で「前回値」を覚えて差分を計算する

よくある場面： センサーの温度変化量を計算する、前回のAPIレスポンスと比較する、電力量計の増分を求めるなど。そのノード専用の記憶域としてcontext変数が役立ちます。

フロー構成：

温度=25 → 差分計算 → 結果

ポイント：

  • Injectノードには異なる温度値を持つボタンを複数用意（25℃／28℃／23℃）
  • Functionノード内で context.get('lastValue') で前回値を取り出す
  • 現在値と前回値の差分を計算し、上昇／下降／変化なしを判定
  • context.set('lastValue', ...) で今回の値を次回のために保存
  • context変数はこのFunctionノード専用なので、他のノードからは見えない

Functionノードのコード：

let previous = context.get('lastValue'); let current = msg.payload; if (previous === undefined) { msg.payload = { current: current, message: "初回測定です" }; } else { let diff = current - previous; msg.payload = { current: current, previous: previous, difference: diff, trend: diff > 0 ? "上昇" : diff < 0 ? "下降" : "変化なし" }; } context.set('lastValue', current); return msg;

💡 3つの例の使い分けまとめ：
  • flow（例1）：同じタブ内の複数ノードで共有（カウンター、状態フラグなど）
  • global（例2）：すべてのタブで共有（設定値、APIキーなど）
  • context（例3）：そのノード内だけで完結（前回値、ノード固有の状態）

📊 5. よくある使い方のパターン

パターン1: ノード内での記憶 → context

そのノード内だけで使う一時データや状態。Functionノード内でのカウンターや前回値の保存など

パターン2: フロー内で共有 → flow

同じタブ内の複数ノードで使いたいデータ。カウンター、最後の値、設定値など

パターン3: 全体で共有 → global

すべてのフローで使いたいデータ。APIキー、ユーザー設定、共通の定数など

パターン4: 独立フロー間の中継 → flow

別々のタイミング・入力経路で来るセンサー値を、1本のフローで合算したい時の中継点

⚠️ 注意： global変数は便利ですが、使いすぎるとフローが複雑になります。本当に必要な場合だけ使用しましょう。まずは flow変数で足りないか検討してください。

🏋️ 6. 演習問題

実際に手を動かして、flow / context / global 変数の使い方を身につけましょう。各問題には難易度・要求仕様・ヒント・解答例フローが用意されています。

演習1: flow変数で訪問カウンターを作る初級

📝 課題：

Injectノードを押すたびに、flow変数 visitCount を +1 して、「◯回目の訪問です」というメッセージを Debug に表示してください。

🎯 要求仕様：

  • Inject、Change、Debug の3ノードを使用(Functionノードは使わない)
  • flow変数名：visitCount
  • ChangeノードのJSONata機能を使って、flow変数を +1 する
  • msg.payload を「1回目の訪問です」のような文字列にする

✅ 成功の条件：

  • 1回目のInject：Debug に「1回目の訪問です」と表示される
  • 2回目のInject：「2回目の訪問です」と表示される
  • 何度押しても順番にカウントアップしていく
  • Node-REDを再起動すると 1 から再スタートする
💡 ヒント

Changeノードの設定（2ルール）：

  • ルール1: 値の代入（Set）
    • 対象: flow.visitCount
    • 値のタイプ: JSONata式
    • 値: ($flowContext("visitCount") ? $flowContext("visitCount") : 0) + 1
  • ルール2: 値の代入（Set）
    • 対象: msg.payload
    • 値のタイプ: JSONata式
    • 値: $flowContext("visitCount") & "回目の訪問です"

※ JSONataの & は文字列連結演算子です。

演習2: 独立した2つのフローの値を合算する初級

📝 課題：

センサーAとセンサーBの値を、それぞれ独立したフロー（別々のInjectチェーン）で受け取り、3つ目のフローで両方の最新値を使って合計と平均を計算してください。実運用では「センサーAとBはバラバラのタイミングで値が更新されるが、計算時には両方の最新値が必要」という場面がよくあります。

🎯 要求仕様：

  • 3つの独立したフロー（すべて同じタブ内）を作成
    • フロー1: Inject「センサーA=20」→ Change（flow.sensorA に保存）→ Debug
    • フロー2: Inject「センサーB=60」→ Change（flow.sensorB に保存）→ Debug
    • フロー3: Inject「計算実行」→ Change（flow.sensorA と flow.sensorB から合計・平均を計算）→ Debug
  • センサーA と センサーB は別々のタイミングで押しても動作する（= flow変数で値が保持される）
  • 計算実行時の出力は { "A": 20, "B": 60, "合計": 80, "平均": 40 } のJSON
  • Functionノードは使わず、Changeノードだけで実装

✅ 成功の条件：

  • センサーAを押す → Debug に「センサーA=20 を保存しました」と表示
  • センサーBを押す → Debug に「センサーB=60 を保存しました」と表示
  • 計算実行を押す → Debug に {A: 20, B: 60, 合計: 80, 平均: 40} が表示
  • センサーAの値を別の数値に変えて先に更新 → 計算実行すると最新値で計算される
  • ABの取得順番（A→B でも B→A でも、片方だけ更新してから計算してもOK）に依存しない
💡 なぜ flow変数なのか？

msg.payload だけで受け渡そうとすると、AとBが「同じ1本のフロー」に並んでいないと値が揃いません。ところがAとBは別々のタイミング・別々の入力経路から来るので、それぞれの値を「どこかに置いておく」必要があります。ここで flow変数が中継地点の役割を果たします。

💡 ヒント

センサーA保存のChangeノード（2ルール）：

  • ルール1: flow.sensorA を msg.payload に設定（型: msg）
  • ルール2: msg.payload を JSONata "センサーA=" & $flowContext("sensorA") & " を保存しました"

センサーB保存のChangeノード：同上で sensorA → sensorB に読み替え

計算のChangeノード（1ルール）：

  • ルール: msg.payload を JSONata式 に設定
  • 式:
    { "A": $flowContext("sensorA"), "B": $flowContext("sensorB"), "合計": $flowContext("sensorA") + $flowContext("sensorB"), "平均": ($flowContext("sensorA") + $flowContext("sensorB")) / 2 }

Injectノードの設定：

  • センサーA: payload = 20（数値型）
  • センサーB: payload = 60（数値型）
  • 計算実行: payload = 空（タイムスタンプでOK）

演習3: global変数でユーザー設定を保存・参照する中級

📝 課題：

Injectで送った「ユーザー名」をglobal変数に保存し、別のInjectで挨拶メッセージを作るフローを作成してください。global変数を使うことで、設定値がフロー全体で共有されることを体験します。

🎯 要求仕様：

  • Injectノード2つ、Changeノード2つ、Debugノード1つ（または複数）を使用
  • 1つ目のフロー：Inject「名前を設定」→ Change → Debug
    • msg.payload に入った文字列「Alice」を global.userName に保存
  • 2つ目のフロー：Inject「挨拶」→ Change → Debug
    • global.userName を取り出して、「こんにちは、Alice さん！」を msg.payload に設定

✅ 成功の条件：

  • 「名前を設定」Injectを押す → Debug に「名前 Alice を保存しました」と表示
  • 「挨拶」Injectを押す → Debug に「こんにちは、Alice さん！」と表示
  • 別のタブ（フロー）に「挨拶」フローを移しても同じように動作する（= global変数の恩恵）
💡 ヒント

1つ目のChangeノード（名前保存）：

  • ルール1: global.userName を msg.payload に設定（型: msg）
  • ルール2: msg.payload を JSONata "名前 " & $globalContext("userName") & " を保存しました"

2つ目のChangeノード（挨拶生成）：

  • ルール: msg.payload を JSONata "こんにちは、" & $globalContext("userName") & " さん！"

Injectノードの設定：

  • 1つ目: payload = "Alice"（文字列型）
  • 2つ目: payload は空（タイムスタンプでOK）

演習4: context変数で直近3回の移動平均を計算する上級

📝 課題：

Functionノード内でcontext変数を使って、直近3回の値の平均（移動平均）を計算するフローを作成してください。ノイズの多いセンサー値をなめらかにする実用的な処理です。

🎯 要求仕様：

  • Inject（数値送信）→ Function（移動平均） → Debug の構成
  • Functionノード内で context.get('history') / context.set('history', ...) を使用
  • 直近3回までの値を配列で保持する（3回を超えたら古いものを削除）
  • 出力は { current: 現在値, history: 履歴配列, average: 平均値 } のJSON
  • Injectは3つ以上用意（例：10, 20, 30, 40, 50）

✅ 成功の条件：

  • 1回目のInject(10)：history=[10]、average=10
  • 2回目のInject(20)：history=[10, 20]、average=15
  • 3回目のInject(30)：history=[10, 20, 30]、average=20
  • 4回目のInject(40)：history=[20, 30, 40]（10が押し出される）、average=30
  • 5回目のInject(50)：history=[30, 40, 50]、average=40
💡 ヒント

Functionノードの実装手順：

  1. context.get('history') || [] で履歴配列を取得（無ければ空配列）
  2. history.push(msg.payload) で現在値を追加
  3. if (history.length > 3) history.shift() で4件目以降は先頭を削除
  4. history.reduce((a, b) => a + b, 0) / history.length で平均を計算
  5. context.set('history', history) で次回用に保存
  6. msg.payload に結果JSONをセットして return

注意： context.get で取得した配列を直接 push で変更しても、context.set で明示的に保存し直すのが安全です。

📋 7. チートシート

変数の種類 スコープ（共有範囲） Changeノードでの指定 Functionノードでの指定 典型的な用途
context そのノード内のみ context.xxx context.get() / set() 前回値、ノード内カウンター
flow 同じタブ内のすべてのノード flow.xxx flow.get() / set() 共有カウンター、センサー値中継
global すべてのタブ（フロー） global.xxx global.get() / set() APIキー、全体設定

JSONata関数（Changeノード等で使用）

関数 用途 例
$flowContext("キー") flow変数の値を読み取る $flowContext("counter") + 1
$globalContext("キー") global変数の値を読み取る $globalContext("threshold")
& 文字列連結（JavaScriptの + に相当） "カウント: " & $flowContext("counter")

🎓 まとめ

📌 Part 2 の要点：
  • context系変数 = フローを超えて値を覚えておく仕組み
  • スコープは context（ノード） / flow（タブ） / global（全体） の3種類
  • 読み書きは Changeノード（GUI）でも Functionノード（JavaScript）でも可能
  • 迷ったら flow を使う。global は本当に全タブで共有が必要な時だけ
  • Node-RED再起動で消える。永続化したい場合は別途設定が必要
