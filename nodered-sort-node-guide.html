このガイドでは、Node-REDのSortノードについて、初心者の方でも理解できるように説明します。

1. Sortノードとは?

「Sort」って何?

Sortノードは、配列やメッセージシーケンスを並べ替えるノードです。 身近な例でいうと、図書館の司書が本を著者名順やタイトル順に棚へ並べ替えるような作業にあたります。

図書館の本の整理に例えると

  • 📖 バラバラに積まれた本 = ソートされていない配列
  • 🔤 タイトルのABC順で並べる = 昇順（ascending）ソート
  • 🔢 出版年の新しい順で並べる = 降順（descending）ソート
  • 📑 著者名で並べ、同じ著者は発売日順 = 複合キーでのソート

基本的な動作

Inject → Template → Sort → Debug

Sortノードは、配列またはメッセージシーケンスを受け取り、指定した条件で並べ替えて出力します。 文字列順、数値順、オブジェクトのプロパティ順など、さまざまな方法でソートできます。

入力（未ソート）
["orange", "banana", "apple", "kiwi"]
↓ Sort（昇順） ↓
出力（ソート済み）
["apple", "banana", "kiwi", "orange"]

2. Sortノードの2つのモード

配列 配列ソート

msg.payloadの配列を並べ替え

配列の要素を 直接ソート

シーケンス メッセージシーケンス

Splitで分割したメッセージを並べ替え

msg.partsを使用して メッセージ順序を変更

設定項目一覧

Sort ノードの設定は、何を・何を基準に・どの順で並べ替えるかを決める。表の並びはエディターの項目順に合わせている。

設定項目 説明 既定値
対象 並べ替えるデータを指定する。配列（msg.payload など）か、split で分割したメッセージシーケンスを選べる。 msg.payload
キー 並べ替えの基準となる値。配列の各要素そのもの、または要素内のプロパティを JSONata 式で指定する。 要素の値
順序 並び順。「昇順」か「降順」。 昇順
数値として比較 値を数値とみなして並べ替える。オフのときは文字列として比較するため、"10" が "9" より前になる。 オフ
名前 エディター上のノード表示名。 （空）

文字列ソートと数値ソートの違い

同じデータでも「文字列として」と「数値として」でソート結果が異なります。

入力
["1024", "86", "256", "100", "9"]
↓ ソート ↓
文字列順
["100", "1024", "256", "86", "9"]

数値順
["9", "86", "100", "256", "1024"]

文字列順と数値順の違い

  • 文字列順: 先頭文字から辞書順で比較（"1024" は "256" より前）
  • 数値順: 数値の大小で比較（9 < 86 < 100 < 256 < 1024）

パターン1: 配列の昇順ソート（文字列）

用途: 文字列配列をアルファベット順に並べ替え

Inject → Template → Sort
(昇順) → Debug

動作の流れ

  1. ["orange", "banana", "apple", "kiwi"] を入力
  2. Sortノードが文字列として昇順ソート
  3. ["apple", "banana", "kiwi", "orange"] を出力

設定例:

対象: msg.payload 順序: 昇順 数値として比較: OFF 入力: ["orange", "banana", "apple", "kiwi"] 出力: ["apple", "banana", "kiwi", "orange"]

パターン2: 配列の降順ソート

用途: 配列を逆順に並べ替え

Inject → Template → Sort
(降順) → Debug

設定例:

対象: msg.payload 順序: 降順 数値として比較: OFF 入力: ["orange", "banana", "apple", "kiwi"] 出力: ["pear", "orange", "kiwi", "banana", "apple"]

パターン3: 数値としてソート

用途: 数値を含む配列を数値順に並べ替え

Inject → Template → Sort
(数値) → Debug

数値ソートのポイント

  • 「数値として比較」にチェックを入れると、文字列でも数値として比較
  • "9" は "100" より小さいと判定される（文字列順だと逆）

設定例:

対象: msg.payload 順序: 昇順 数値として比較: ON 入力: ["1024", "86", "256", "100", "9"] 出力: ["9", "86", "100", "256", "1024"]

パターン4: オブジェクト配列のソート（JSONata式）

用途: オブジェクトの特定プロパティで並べ替え

Inject → Template
(商品配列) → Sort
(price) → Debug

JSONata式でのソート

  • キーに price を指定すると、各要素のpriceプロパティでソート
  • 配列要素がオブジェクトの場合に使用

設定例:

対象: msg.payload キー: price (JSONata式) 順序: 昇順 入力: [ {"name": "orange", "price": 80}, {"name": "banana", "price": 250}, {"name": "apple", "price": 100} ] 出力: [ {"name": "orange", "price": 80}, {"name": "apple", "price": 100}, {"name": "banana", "price": 250} ]

パターン5: 複合キーでのソート

用途: 複数の条件で並べ替え（第1キー、第2キー）

Inject → Template → Sort
(price+name) → Debug

設定例:

対象: msg.payload キー: $substring("0000" & $string(price), -4) & name 順序: 昇順 この式の意味: 1. priceを4桁のゼロ埋め文字列に変換（100 → "0100"） 2. nameを連結（"0100apple"） 3. この連結文字列でソート 結果: 価格順、同じ価格なら名前順

パターン6: メッセージシーケンスのソート

用途: Splitで分割したメッセージを並べ替え

Inject → Template → Split →→→ Sort
(シーケンス) →→→ Debug

メッセージシーケンスのポイント

  • Splitノードで分割されたメッセージにはmsg.partsが付与される
  • Sortノードはmsg.partsを使ってメッセージの順序を変更
  • 出力もメッセージシーケンス（複数メッセージ）として出力

設定例:

対象: メッセージシーケンス キー: msg.payload 順序: 昇順 入力シーケンス: "orange" → "banana" → "apple" → "kiwi" 出力シーケンス: "apple" → "banana" → "kiwi" → "orange"

4. 実践演習

演習1: 果物名のソート初級

課題

果物名の配列を降順（Z→A）に並べ替えてください。

要求仕様

  • Inject、Template、Sort、Debugノードを使用
  • 入力: ["grape", "apple", "cherry", "banana"]
  • 出力: ["grape", "cherry", "banana", "apple"]

期待される動作

  • Injectボタンをクリック → Debugにソートされた配列が表示

成功の条件

  • デバッグパネルに1つのメッセージとして配列が表示される
  • 出力配列が ["grape", "cherry", "banana", "apple"] の順になっている（Z→A降順）
  • 配列の先頭が "grape"（アルファベット最後）、末尾が "apple"（アルファベット最初）になっている
💡 ヒント

Injectノードの設定:

  • payload: タイムスタンプ（デフォルト）

Templateノードの設定:

  • 形式: JSON
  • テンプレート: ["grape", "apple", "cherry", "banana"]
  • 出力: JSONオブジェクト

Sortノードの設定:

  • 対象: msg.payload
  • 順序: 降順
  • 数値として比較: OFF（チェックなし）

演習4: メッセージシーケンスのソート上級

課題

Splitで分割したメッセージシーケンスをソートし、Joinで再結合してください。

要求仕様

  • Inject、Template、Split、Sort、Join、Debugノードを使用
  • 入力: 改行区切りの都市名リスト
  • 出力: アルファベット順に並べ替えられた配列

期待される動作

  • Splitで分割 → Sortで並べ替え → Joinで配列に再結合

成功の条件

  • デバッグパネルに1つのメッセージとして配列が表示される
  • 出力配列が ["Kyoto", "Nagoya", "Osaka", "Tokyo"] のアルファベット昇順になっている
  • 配列の先頭が "Kyoto"、末尾が "Tokyo" になっている
  • Joinによってメッセージシーケンスが1つの配列に再結合されている
💡 ヒント

Templateノードの設定:

  • 形式: テキスト
  • テンプレート: Tokyo, Osaka, Kyoto, Nagoyaを改行で区切って入力
  • 出力: 文字列

Splitノードの設定:

  • 分割: 改行（\n）

Sortノードの設定:

  • 対象: メッセージシーケンス
  • キー: msg.payload
  • 順序: 昇順

Joinノードの設定:

  • モード: 自動（または配列形式）

5. まとめ

Sortノードの重要ポイント

  • 2つのモード: 配列ソートとメッセージシーケンスソート
  • 順序: 昇順（ascending）と降順（descending）
  • 数値モード: 数値として比較するか文字列として比較するか選択可能
  • JSONata式: オブジェクト配列のプロパティでソート可能
  • Split/Joinとの連携: メッセージシーケンスのソートはSplit/Joinと組み合わせて使用

よくある間違い

  • 文字列数値の混同: "9" が "100" より大きいと判定される（文字列順）
  • JSONata式の指定ミス: 配列要素のプロパティ名を正確に指定する
  • シーケンスモードの選択忘れ: Splitからのメッセージは「メッセージシーケンス」を選択
  • msg.partsの欠落: シーケンスソートにはmsg.partsが必要

次のステップ

Sortノードをマスターしたら、以下のノードも学んでみましょう:

  • Splitノード: 配列やテキストを分割
  • Joinノード: メッセージを結合
  • Batchノード: メッセージをグループ化

6. トラブルシューティング

よくある問題と解決方法

問題 原因 解決方法
ソートされない 配列でない値が入力されている 入力データが配列であることを確認
数値順にならない 「数値として比較」がOFF 「数値として比較」をONに設定
シーケンスソートが動かない msg.partsがない Splitノードを経由させる
JSONata式でエラー プロパティ名のスペルミス オブジェクトのプロパティ名を確認
結果が配列でなくバラバラ シーケンスモードで使用 Joinノードで再結合する
順序が安定しない 同じ値の要素の順序 複合キーで第2ソート条件を追加

ケース1: IoTセンサーデータのランキング

複数センサーから温度データ受信 ↓ Sortノード（数値降順） ↓ 最も高温のセンサーが先頭 ↓ アラート判定・通知

複数ソースからログ収集 ↓ Sortノード（timestamp昇順） ↓ 時系列順に整列 ↓ データベース保存
