🧮 Node-RED JSONata式ガイド

JSONata は、Node-REDで「値を動的に計算・加工」するために使う式言語です。Changeノードで式を書くときに出てくる「J:」アイコンで選択できます。このガイドでは、Changeノードを中心に、JSONataの基本から実践パターンまでを日本語で解説します。

📖 1. JSONataとは？ どこで使える？

🤔 JSONataはなぜ必要か

Changeノードの「値の代入」で、文字列や数値を固定値で入れるだけなら JSONata は不要です。しかし実務では、「現在時刻を付ける」「payload を 2 倍する」「文字列を連結する」「前回値と比較する」など、動的に計算した値を入れたい場面が頻繁に出てきます。これを実現するのが JSONata 式です。

❌ JSONataなし（固定値）

msg.payload = "Hello" msg.topic = "greeting"

同じ値しか出せない

✅ JSONataあり（動的）

msg.payload = "Hello, " & msg.user & "!" msg.topic = "greeting-" & $now()

入力や状況に応じて値を計算できる

🎯 Node-REDでJSONataが使えるノード

Changeノード以外にも「J:」アイコン（式セレクタ）が出る場所では同じ JSONata 文法が使えます。このガイドでは主にChangeノードを題材にしますが、文法は共通なので他ノードでも応用できます。

ノード JSONataが使える場所 用途
Change ← 主役 値の代入の「値のタイプ」で選択 動的に値を計算して設定
Switch 条件の「値の型」で選択 式で条件判定（例: payload > 30）
Template テンプレートモードを「JSONata」に テンプレート全体を式として評価
Debug 「対象」で式を選択 特定の計算結果だけをデバッグ表示
Inject payloadの「値のタイプ」で選択 起動時に式で値を生成
📌 このガイドの方針：
  • 第2〜7章はChangeノード内で JSONata を使う前提で解説
  • 第8章で他ノードでの使い方の違いを補足
  • 第9章に実践パターン集（インポート可能なフロー付き）
  • 第10章にチートシート（リファレンス用）

🚀 2. 最初のJSONata（Changeノードで試す）

いきなり難しい関数から入らず、まずは「payload を参照する」「+1 する」「文字列を連結する」の3ステップで JSONata の雰囲気に慣れましょう。

ステップ1: payload をそのまま取り出す

Changeノードの「値の代入」で、値のタイプを J: 式 にすると、式の中では payload と書くだけで msg.payload を参照できます（msg. は省略される）。

msg.payload = 25
→
JSONata式: payload → 25

ステップ2: 計算する

JavaScriptと同じ四則演算が使えます。

payload * 2
msg.payload = 25
→
式の結果: 50

ステップ3: 文字列を連結する

JavaScriptでは + を使いますが、JSONataでは & を使います。ここが最初の落とし穴です。

"温度は " & payload & "℃ です"
msg.payload = 25
→
式の結果: "温度は 25℃ です"
⚠️ JavaScriptとの違い（最重要）：
  • 文字列連結は + ではなく &
  • 比較は === ではなく =（1文字）
  • 不等価は !== ではなく !=
  • 論理演算子は && / || ではなく and / or

📝 3. 文字列の操作

JSONataには文字列を操作する豊富な関数が用意されています。ここでは Changeノードでよく使うものを紹介します。

🔗 文字列の連結

式 意味 結果（payload="Alice" の場合）
"Hello, " & payload 前に固定文字列を連結 "Hello, Alice"
payload & " さん" 後ろに固定文字列を連結 "Alice さん"
"[" & topic & "] " & payload 複数プロパティを連結 "[greeting] Alice"（topicが"greeting"の場合）

🔤 よく使う文字列関数

関数 意味 例と結果
$length(str) 文字数を返す $length("Hello") → 5
$uppercase(str) 大文字化 $uppercase("abc") → "ABC"
$lowercase(str) 小文字化 $lowercase("ABC") → "abc"
$trim(str) 前後の空白を削除 $trim(" hi ") → "hi"
$substring(str, start, len) 部分文字列を取得 $substring("Hello", 0, 3) → "Hel"
$substringBefore(str, sep) 区切り文字の前を取得 $substringBefore("a.b.c", ".") → "a"
$substringAfter(str, sep) 区切り文字の後を取得 $substringAfter("a.b.c", ".") → "b.c"
$split(str, sep) 区切って配列化 $split("a,b,c", ",") → ["a","b","c"]
$join(arr, sep) 配列を文字列に結合 $join(["a","b"], "-") → "a-b"
$replace(str, old, new) 置換 $replace("abc", "b", "X") → "aXc"
$contains(str, sub) 含まれるか（真偽値） $contains("Hello", "ell") → true
$string(value) 数値などを文字列に変換 $string(25) → "25"
$pad(str, width) 指定幅に空白でパディング $pad("5", 3) → " 5"
💡 よくある実例：
$uppercase($substringBefore(payload, "@"))

メールアドレスのローカルパートを大文字化。"alice@example.com" → "ALICE"

🔢 4. 計算・数値操作

🧮 四則演算と基本演算子

演算子 意味 例
+ - * / 四則演算 payload * 1.8 + 32（摂氏→華氏）
% 剰余 payload % 10
= != 等価・不等価（1文字） payload = 100
< > <= >= 大小比較 payload >= 30
and or 論理演算（&&/||ではない） payload > 0 and payload < 100
? : 三項演算子（JavaScriptと同じ） payload > 30 ? "高温" : "正常"

📐 よく使う数値関数

関数 意味 例と結果
$number(value) 文字列→数値変換 $number("25") → 25
$abs(n) 絶対値 $abs(-5) → 5
$floor(n) 切り下げ $floor(3.7) → 3
$ceil(n) 切り上げ $ceil(3.2) → 4
$round(n, precision) 四捨五入（precision省略可） $round(3.456, 1) → 3.5
$power(base, exp) 累乗 $power(2, 10) → 1024
$sqrt(n) 平方根 $sqrt(9) → 3
$sum(arr) 配列の合計 $sum([1,2,3]) → 6
$average(arr) 配列の平均 $average([1,2,3]) → 2
$max(arr) / $min(arr) 最大値 / 最小値 $max([3,1,5,2]) → 5
$random() 0〜1の乱数 $random() → 0.42...
💡 よくある実例（温度変換）：
$round(payload * 1.8 + 32, 1)

摂氏→華氏に変換して小数1桁に丸める。25 → 77、20.5 → 68.9

📦 5. 配列・オブジェクト操作

配列の基本

式 意味 例（payload=[10,20,30]の場合）
payload[0] 最初の要素 10
payload[-1] 最後の要素（便利！） 30
$count(payload) 要素数 3
$reverse(payload) 逆順 [30, 20, 10]
$sort(payload) 昇順ソート [10, 20, 30]
$append(payload, [40]) 要素を追加した新配列 [10, 20, 30, 40]

配列の加工（map / filter）

JavaScriptの .map() や .filter() に似た処理が書けます。

// 例: payload = [{"name":"A", "temp":25}, {"name":"B", "temp":32}, {"name":"C", "temp":18}] payload.name → ["A", "B", "C"] // 各要素のnameを取り出す（ドット記法で暗黙のmap） payload[temp > 20] → [{"name":"A", "temp":25}, {"name":"B", "temp":32}] // 条件でフィルタ $sum(payload.temp) / $count(payload) → 25 // 温度の平均

オブジェクトの構築

JSONata式内で直接オブジェクトを組み立てられます。これがJSONataの強力さの核心です。

{"name": payload, "timestamp": $now(), "unit": "℃"}
msg.payload = 25
→
{"name": 25, "timestamp": "2026-04-19T...", "unit": "℃"}
💡 センサーデータを整形する典型例：
// 入力 msg.payload = {"raw_temp": 2530, "raw_humid": 650} // 100で割って実数に戻す { "temperature": payload.raw_temp / 100, "humidity": payload.raw_humid / 10, "time": $now() }

🗃️ 6. context変数の読み書き

Changeノード内の JSONata 式から、flow変数や global変数を読み取ることができます。書き込みは Change ノードの「対象」側で flow/global を選びます。

読み取り: $flowContext / $globalContext

関数 用途 例
$flowContext("キー") flow変数の値を取得 $flowContext("counter")
$globalContext("キー") global変数の値を取得 $globalContext("apiKey")

書き込み: ChangeノードのUI操作

式ではなく、Changeノードの「値の代入」ルールで「対象」側を flow または global にします。

ルール: 値の代入 対象: flow.counter ← ここが「flow」タイプ 値のタイプ: JSONata式 値: $flowContext("counter") + 1 （解説: 現在値を取って+1し、flow.counterに書き戻す）

典型パターン: カウンター

JSONataでは「式」と「書き込み先」は別なので、カウンターを作るには「flow変数を読んで +1 した結果を、同じ flow変数に書き戻す」という形になります。

Changeノード（2ルール）: ルール1: flow.count を JSONata式 に設定 式: ($flowContext("count") ? $flowContext("count") : 0) + 1 （初回は undefined なので 0 として扱う） ルール2: msg.payload を JSONata式 に設定 式: $flowContext("count") （ルール1の結果を読んで出力）
📌 なぜ三項演算子（? :）を使うのか：

初回実行時は $flowContext("count") が undefined になり、undefined + 1 はエラーになります。そのため 値が存在するかチェック → 無ければ0として扱う という防御が必要です。

💡 context変数の詳細は： 変数ガイド Part 2（context系変数） に完全なフロー例・演習問題があります。

📅 7. 日時・環境変数

日時関数

関数 意味 結果の例
$now() 現在時刻のISO文字列 "2026-04-19T10:30:00.000Z"
$millis() 現在時刻のUnixミリ秒 1744632600000
$fromMillis(ms) ミリ秒→ISO文字列 $fromMillis(1744632600000) → "2026-04-19T..."
$toMillis(iso) ISO文字列→ミリ秒 $toMillis("2026-04-19") → 1744...
💡 タイムスタンプ付与の典型例：
{"data": payload, "received_at": $now()}

センサー値に受信時刻を付けて構造化。ログ用途でよく使う。

環境変数

JSONata式内から $env("NAME") で環境変数を読めます（Changeノードの「値のタイプ」で env を選ぶ方法もあるが、式内で組み合わせるならこちら）。

関数 意味 例
$env("NAME") 環境変数の値を取得 $env("API_KEY") → "xxx"
// 例: APIキーを付けたリクエストペイロードを作る { "key": $env("API_KEY"), "query": payload }

型チェック・存在チェック

関数 意味 例
$exists(value) undefined/null でないか $exists(payload.user) → true/false
$type(value) 型名を返す $type(payload) → "number" 等
$boolean(value) 真偽値に変換 $boolean("true") → true

🔀 8. Change以外のノードでの使い方

JSONataの文法はどのノードでも共通ですが、「式の結果がどう使われるか」がノードによって異なります。ここでは代表的な3つを紹介します。

🎛️ Switchノード: 式で条件判定

Switchノードの条件欄で「値の型」を JSONata式 にすると、式の結果（真偽値）で分岐できます。

Inject
payload → Switch
payload > 30 → Debug
Switchノード条件: 条件1: JSONata式 → payload > 30 （高温ルート） 条件2: JSONata式 → payload <= 30 （正常ルート）

📝 Templateノード: 式の結果を埋め込む

Templateノードのテンプレート内で {{expression}} を使うと、JSONata式の結果が埋め込まれます（Mustache構文＋JSONata）。

// Templateノードのテンプレート {{payload.user}} さん、温度は {{payload.temp}}℃ です。 記録時刻: {{$now()}} // 出力例 Alice さん、温度は 25℃ です。 記録時刻: 2026-04-19T10:30:00.000Z

🐛 Debugノード: 特定の計算結果だけ出す

Debugノードの「対象」を JSONata式 にすると、msgの一部を計算した結果だけをサイドバーに表示できます。

Debugノード対象: JSONata式 式: $count(payload) → msg.payload が配列の時、要素数だけを表示
💡 共通のルール：
  • JSONata式内で payload / topic と書くと、暗黙で msg.プロパティ を指す
  • 関数名は常に $関数名() の形（$ 前置）
  • 文字列は ダブルクォート推奨（シングルも使えるが混乱回避のため）

パターン3: オブジェクト配列から条件にマッチするものだけ抽出

入力： [{"name":"A","temp":25},{"name":"B","temp":32},{"name":"C","temp":18}]

式（温度30℃以上のものだけ）：

payload[temp >= 30]

結果： [{"name":"B","temp":32}]

パターン4: 日時を人間が読める形式に整形

JSONataの $fromMillis() に format引数を指定すると日時を好きな書式にできます。

$fromMillis($millis(), "[Y]-[M01]-[D01] [H01]:[m01]:[s01]") → "2026-04-19 10:30:45"
💡 書式記号（よく使うもの）：
  • [Y] 年 / [M01] 月（2桁） / [D01] 日（2桁）
  • [H01] 時 / [m01] 分 / [s01] 秒
  • [F] 曜日（Monday）/ [FNn,*-3] 曜日3文字（Mon）

🎓 まとめ

📌 JSONataの要点：
  • JSONataは値を動的に計算・加工するための式言語
  • Changeノード内が主役だが、Switch/Template/Debug/Injectでも共通の文法で使える
  • JavaScriptとは演算子が一部違う（&、=、and/or、[-1]）
  • payload と書くだけで msg.payload を指す（暗黙の省略）
  • context変数は $flowContext() / $globalContext() で読み取る
  • オブジェクト {...} や配列 [...] を式の中で直接構築できるのが強力
