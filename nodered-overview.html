💡 Node-REDとは？

Node-REDは、ブラウザ上で動作するビジュアルプログラミングツールです。 「ノード」と呼ばれる部品を線でつなぐだけで、複雑なプログラムを簡単に作成できます。

🖱️
ドラッグ&ドロップ

マウス操作だけでプログラミング。コードを書く必要はほとんどありません。

🔌
IoTに最適

センサーやデバイスとの連携が簡単。Raspberry Piとの相性も抜群です。

🌐
API連携

WebサービスやクラウドAPIとの接続も簡単に実現できます。

⚡
リアルタイム

変更したらすぐにデプロイ。動作確認がスピーディに行えます。

🖥️ エディタ画面の構成

Node-REDのエディタは主に4つのエリアで構成されています。実際の画面を見てみましょう。

画面構成の詳細説明

① パレット（左側）

使用可能なノードが一覧表示されます。

  • 検索バー： ノードを名前で検索
  • 共通カテゴリ： inject（入力）、debug（デバッグ）など
  • 機能カテゴリ： function（関数）、switch（分岐）など
  • 色分け： ノードの種類ごとに色が違う

② ワークスペース（中央）

フローを作成するメインの作業エリアです。

  • グリッド背景： ノードを整列しやすい
  • タブ表示： 複数のフローを切り替え可能
  • ドラッグ&ドロップ： パレットからノードを配置
  • 接続： ノード間を線でつなぐ

③ デバッグパネル（右側）

debugノードの出力を表示するエリアです。

  • リアルタイム表示： データの流れを確認
  • フィルター機能： 特定のフローのみ表示
  • メッセージ詳細： JSONデータの構造を確認
  • クリアボタン： 表示内容をリセット

④ デプロイボタン（右上）

作成したフローを実行するためのボタンです。

  • デプロイ： 変更を反映して実行開始
  • 変更検知： 未保存の変更があると色が変わる
  • 部分デプロイ： 変更箇所のみ更新も可能
  • ショートカット： Ctrl+D（Windows）/ Cmd+D（Mac）

🧩 主なノードの種類

ノードは機能によって色分けされています。

inject, http in, mqtt in

処理系（青）

function, change, switch

出力系（緑）

debug, http response

変換系（オレンジ）

json, csv, template

📋 フローを整理する・部品化するテクニック

ノードが増えてきたら、以下の機能でフローを「読みやすく」「使い回せる」状態に育てていきましょう。

🔲 ノードのグループ化 ─ まずはこれだけでOK

💡 イメージは PowerPoint や Excel の図形グループ化と同じです。関連するノードを枠で囲んでひとまとめにし、まとめて移動・コピー・削除できます。

基本操作

  • 作成：ドラッグまたは Shift+クリックでノードを複数選択 → 右クリック「Group selection」/ Ctrl+Shift+G（Mac は ⌘+Shift+G）
  • 解除：Ctrl+Shift+U / ⌘+Shift+U
  • 編集：グループの枠をダブルクリック → 名前・背景色・枠線色を変更
  • まとめて移動：枠をドラッグすると中のノードが一緒に動く
  • 入れ子：グループの中にさらにグループを作れる（Node-RED 3.1 以降）

「センサー入力部」「データ加工部」「出力部」のように 処理の役割ごとに色分け すると、半年後の自分が読んでも理解しやすくなります。

🚀 上級者向け：PowerPoint / Excel にはない追加機能
  • 中のノードをまとめて有効化 / 無効化：グループの枠を右クリック →「Enable/Disable selected nodes（選択ノードを有効化/無効化）」で、中の全ノードの実行を一括で止めたり再開できる（デバッグや一時切り替えに便利）。
    ※ グループ自体に「無効」プロパティがあるわけではなく、グループ選択 = 中の全ノード選択という挙動を利用したものです。
  • グループごとエクスポートして再利用：グループを選択した状態でエクスポート（Ctrl+E）すると、中のノードもまとめて JSON として書き出せる。別フロー・別環境にコピペで持ち込める（Git 管理も可能）
  • 環境変数のスコープ：グループに環境変数を定義すると、中のノードからだけ参照できる「専用変数」になる。同じグループを複数配置して、それぞれ違うパラメータで動かせる（Node-RED 2.1 以降）
  • 枠とラベルが常時表示：PowerPoint / Excel の図形グループ化は「選択中だけ枠が出る」が、Node-RED は枠と名前が常に見えるのでドキュメント代わりになる
  • 枠の見た目を細かくカスタマイズ：背景色・枠線色・文字色・ラベル位置を個別に設定できる。Ctrl+Shift+C / Ctrl+Shift+V で他のグループへスタイルをコピー＆ペーストも可能

※ グループ化は 見た目と操作の単位 であって、メッセージの流れや実行順序には影響しません。グループ自体をワイヤで接続することもできません（接続は従来どおりノード同士）。

🧩 サブフロー ─ 同じ処理を「カスタムノード」にする

サブフローは、複数のノードをまとめて 1 つのカスタムノード としてパレットに登録できる機能です。同じ処理を何度も使い回したいときに使います。

こんなときに便利

  • 「センサー値を ℃ → ℉ に変換 + 小数点丸め + 異常値除外」など、複数ノードで実現する処理を 何度も使い回したい
  • 同じロジックを複数のフローで使っており、1 箇所直したら全部に反映したい
  • 複雑な処理を 1 個のノードに見せて フロー全体を見やすくしたい

基本操作

  • 作成：ノードを複数選択 → メニュー「サブフロー」→「選択範囲をサブフロー化」
  • 使う：パレット左側の「subflows」カテゴリーから、通常のノードと同じようにドラッグ
  • 編集：サブフローノードをダブルクリックして中身を編集 → 全インスタンスに反映
  • プロパティ・環境変数：使う側で値を変えられるパラメータを定義可能

📊 グループ化 vs サブフロー の使い分け

🔲 グループ化 🧩 サブフロー
目的見た目の整理部品化・再利用
PowerPoint / Excel で例えると図形のグループ化スライドマスター / マクロ・ユーザー定義関数
複製したら独立した別のノード群になる同じ実体を参照（1 箇所直すと全部反映）
パレット登録されないされる（カスタムノード扱い）
入出力定義しない入出力ポートを定義する
難易度★☆☆ 初心者からOK★★☆ 中級者向け
まず使うべき場面フローが見づらくなってきた同じ処理を 2 回以上書いた

📝 その他の整理機能

  • コメントノード：フロー上にメモを残せる → コメントノードガイド
  • Link ノード：離れた場所のノード同士を線なしでつなげる → Link ノードガイド
  • 複数のフロータブ：画面上部の「+」で新規タブを追加し、機能ごとに分割

📦 メッセージ（msg）オブジェクト

Node-REDでは、ノード間を「msg」というオブジェクトがデータを運びます。 郵便物のようなイメージです。

msg オブジェクトの構造 
{
  payload: "データ本体",
  topic: "トピック名",
  _msgid: "一意のID"
}

💡 ポイント

  • msg.payload が最も重要
  • 自由にプロパティを追加可能
  • debugノードで中身を確認
