# 共有サーバー ポート割当表（講師・運用向け）

> **目的**：1台の共有クラウドサーバー（AWS Lightsail 等）で複数の学習者が Node-RED を動かす際、
> **TCP / UDP / MODBUS の「待ち受けポート」が学習者間で衝突しない**よう、各自に固有ポートを割り当てるための運用ドキュメントです。
> 学習者向けの解説は [学習環境ガイド](nodered-learning-environment-guide.html) を参照してください。

---

## なぜポート割当が必要か

共有サーバーでは、全学習者の Node-RED が**1つのOS（＝1つのポート空間）**を共有します。

- **エディタのポート**（1880, 1881…）は学習者ごとに分かれていれば独立して使えます。
- しかし、フロー内で**新たに待ち受けソケットを開く**ノードは、サーバー全体で1つしかないポートを奪い合います：
  - **TCP In（Listen on / サーバー）**
  - **UDP In（受信）**
  - **MODBUS（Modbus-Server ノードで擬似スレーブを立てる場合）**
- 同じポート番号は**先着1名しか使えず**、他の学習者は「アドレス使用中（EADDRINUSE）」エラーになります。

> 送信・接続だけ（http request, tcp request, mqtt client, modbus client など）は待ち受けないので**衝突しません**。

---

## 割当スキーム（推奨）

学習者ごとに番号 **N（1, 2, 3, …）** を振り、次の規則でポートを割り当てます。

| 用途 | 計算式 | 例: N=1 | 例: N=2 | 例: N=3 |
|------|--------|--------|--------|--------|
| Node-RED エディタ | `1880 + N` | 1881 | 1882 | 1883 |
| MODBUS シミュレータ（Modbus-Server） | `10500 + N` | 10501 | 10502 | 10503 |
| TCP / UDP 演習用ポート帯（各自10個） | `11000 + N*10` 〜 `+9` | 11010–11019 | 11020–11029 | 11030–11039 |

- **MODBUS**：サンプルの既定は `10502`。自分の番号の `10500 + N` に変更して使う（502 は特権ポートで共有環境では開けません）。
- **TCP/UDP**：サンプルの `9000` や `8000` をそのまま使わず、自分の**演習用ポート帯**の中から選ぶ（例 N=1 なら 11010, 11011, …）。
- いずれも **1024 以上の非特権ポート**にすること（共有サーバーは一般ユーザー権限のため）。

## 注意事項

- **ファイアウォール**：クラウドのセキュリティグループ（Lightsail のファイアウォール）で、外部からアクセスさせたいポートのみ開放してください。演習を**サーバー内で完結**（`127.0.0.1` 同士）させる場合は、外部開放は不要です。
- **ブロードキャスト / マルチキャスト**：クラウドのネットワークでは基本的に届きません。これらの演習は[ローカルPC / ラズパイ](nodered-local-pi-exercise-guide.html)で実施してください。
- **物理ハードウェア**（MODBUS RTU、GPIO、センサー）：クラウドでは不可。ラズパイ等の実機が必要です。
- 上記スキームは目安です。既存のポート運用に合わせて調整してください。
